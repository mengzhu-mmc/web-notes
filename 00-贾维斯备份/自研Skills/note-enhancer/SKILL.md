# Note Enhancer — 笔记质量管家

## 职责

扫描 `web-notes/`（或 `notes/`）目录，对每篇笔记进行质量评分，输出报告并提供修复建议。

---

## 1. 质量评分引擎（满分 10 分）

对每篇笔记从以下四个维度各打 0-2.5 分，合计 10 分：

| 维度 | 满分 | 评判标准 |
|------|------|---------|
| **时效性** | 2.5 | 笔记创建/更新时间；内容是否涉及已废弃 API（如 `__proto__`、Vue2 Options API 无说明等） |
| **完整性** | 2.5 | 是否包含：标题、代码示例、面试考点、总结/结论 |
| **结构性** | 2.5 | 是否有清晰章节划分（二级标题）；段落是否合理；有无目录 |
| **关联性** | 2.5 | 是否有 `[[双链]]` 或 `[相关笔记](path)` 连接其他笔记 |

**评分档次：**
- 8-10：✅ 优质笔记
- 5-7：🟡 待优化
- 0-4：🔴 需重写

---

## 2. 扫描模式

### 快速扫描（`快速检查`）
只检查：
- 文件大小 < 500 字节 → 疑似空壳
- 最后修改时间 > 180 天 → 疑似过期
- 无代码块（无 ` ``` `）→ 疑似无示例

输出：疑似问题文件列表 + 数量统计

### 深度扫描（`扫描笔记` / `笔记质量报告`）
逐文件读取内容，按四维评分模型打分，生成完整报告。

### 专项扫描（`检查 [目录名]`）
只扫描指定子目录，如 `检查 04-Vue/` 则只分析 `web-notes/04-Vue/` 下的文件。

---

## 3. 触发词映射

| 用户说 | 执行 |
|--------|------|
| 「扫描笔记」「笔记质量报告」 | 深度扫描全库 |
| 「快速检查」 | 快速扫描 |
| 「检查 [目录名]」 | 专项扫描指定目录 |
| 「笔记过期了吗」「知识库维护」 | 快速扫描 + 时效性专项 |

---

## 4. 输出报告格式

```
📊 笔记质量报告 YYYY-MM-DD
总计：X 篇 | 优质(8+)：X | 待优化(5-7)：X | 需重写(<5)：X

🔴 需重写（前3名）
1. 04-Vue/README.md — 分数: 3/10 — 问题: 无实质内容，全是占位符
2. 02-CSS/flex.md   — 分数: 2/10 — 问题: 文件仅 120 字节，无代码示例
3. ...

🟡 待优化（前5名）
1. 03-JS/closure.md  — 分数: 6/10 — 建议: 补充面试考点 + 双链
2. ...

✅ 本次已优化
- (如有自动修复，列出)
```

---

## 5. 自动修复策略

### 快速修复（直接执行，无需确认）
- 补充「面试考点」占位章节（如果完全缺失）
- 补充「参考资料」章节
- 修正明显的格式问题（如标题层级错误）

### 深度重写（询问用户）
对 🔴 需重写的笔记，输出建议内容框架后询问：
> 「XXX.md 评分 X/10，需要深度优化。要我现在帮你重写吗？」

---

## 6. 定期巡检（配合 HEARTBEAT.md）

每周一心跳时自动触发快速扫描：
1. 读取 `memory/heartbeat-state.json` 中 `lastNoteCheck` 字段
2. 如果距上次检查 > 7 天，执行快速扫描
3. 发现 🔴 需重写笔记 → 推送钉钉通知靓仔
4. 更新 `memory/heartbeat-state.json` 中 `lastNoteCheck` 时间戳

---

## 7. 执行步骤（Agent 操作指南）

当触发本 skill 时，按以下步骤执行：

```
1. 确定扫描模式（根据触发词）
2. 使用 exec 命令列出目标目录文件：
   find /root/.openclaw/workspace/web-notes/ -name "*.md" -type f
   （如果 web-notes/ 不存在，改为 notes/）
3. 根据模式：
   - 快速扫描：用 stat + wc -c 检查文件大小和修改时间
   - 深度扫描：逐个 read 文件内容，按四维评分
   - 专项扫描：限定 find 路径
4. 汇总评分，生成报告
5. 对 🔴 文件提供修复建议或直接修复
6. 更新 memory/heartbeat-state.json
```
