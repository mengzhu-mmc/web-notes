# 间隔复习系统 (Spaced Review)

## 核心机制

基于 SM-2 (SuperMemo 2) 算法的间隔复习：
- 新卡片：1 天后复习
- 复习时根据用户评分调整下次间隔
- 评分 0-5 分：0-2 重新学习，3 勉强记住（间隔不变），4 记住（间隔增长），5 轻松记住（间隔大幅增长）

## 数据文件

- **卡片库**：`data/flashcards.json` — 所有卡片及其复习状态
- **复习日志**：`data/review-log.json` — 复习历史记录

若 `data/` 目录不存在则创建。

### flashcards.json 格式

```json
{
  "cards": [
    {
      "id": "js-closure-001",
      "category": "JavaScript",
      "front": "什么是闭包？",
      "back": "函数 + 其词法环境的引用。应用：数据私有化、函数工厂、柯里化。注意内存泄漏风险。",
      "source": "notes/javascript/2026-03-07-js-interview-top20.md",
      "difficulty": 2.5,
      "interval": 1,
      "repetitions": 0,
      "nextReview": "2026-03-14",
      "created": "2026-03-13"
    }
  ]
}
```

### SM-2 算法实现

每次复习后更新卡片参数：

```
输入：quality (0-5 用户评分)
如果 quality >= 3（记住了）:
  如果 repetitions == 0: interval = 1
  如果 repetitions == 1: interval = 6
  否则: interval = round(interval * difficulty)
  repetitions += 1
否则（忘了）:
  repetitions = 0
  interval = 1

difficulty = max(1.3, difficulty + 0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02))
nextReview = today + interval 天
```

## 工作流

### 1. 生成卡片（从 notes/ 自动提取）

触发：用户说「生成卡片」「从笔记生成闪卡」

1. 扫描 `notes/` 目录下所有 .md 文件
2. 对每个知识点提取「问题 → 答案」对
3. 去重（和已有卡片比较）
4. 写入 `data/flashcards.json`
5. 报告：新增 X 张卡片，总共 Y 张

提取规则：
- 标题（##/###）→ 作为问题的线索
- 要点/代码块 → 作为答案内容
- 每个知识点生成 1-3 张卡片（概念/应用/对比）

#### 🧮 算法卡片专项生成（与 algo-coach 联动）

触发：用户说「生成算法卡片」「从算法笔记生成闪卡」

1. 扫描 `web-notes/08-算法与数据结构/3周冲刺记录/` 目录下所有 .md 文件
2. 针对算法笔记生成**专属格式**的卡片：
   - **正面**：「[题目名 / 问题类型] 用什么算法？关键思路是？」
   - **背面**：「算法名 + 核心套路（一句话）+ 时间复杂度 + 关键代码模板要点」
3. 卡片 `category` 标记为 `算法`（区别于普通知识卡片）
4. 去重后写入 `data/flashcards.json`
5. 报告：新增 X 张算法卡片，总算法卡片数 Y

算法卡片格式示例：
```json
{
  "id": "algo-backtrack-001",
  "category": "算法",
  "front": "含重复元素的子集问题用什么算法？关键思路是？",
  "back": "回溯 + 排序去重。核心套路：先排序，递归时同层跳过相同元素（if i > start && nums[i] == nums[i-1] continue）。时间复杂度 O(2^n)。",
  "source": "web-notes/08-算法与数据结构/3周冲刺记录/..."
}
```

### 2. 手动添加卡片

触发：用户说「添加卡片」+ 内容

解析用户输入，创建新卡片加入库中。

### 3. 开始复习

触发：用户说「复习」「今天复习什么」

支持可选参数 `mode: "interview"` — 触发词：「算法复习」「面试模式复习」「只复习算法」
- `mode: "interview"` 时：只筛选 `category` 为 `算法` 或 `面试题` 的卡片（过滤掉 CSS/React 等日常知识卡片）
- 面试临近（距冲刺结束 ≤ 7 天）时，自动提示：「🎯 面试冲刺期，建议用 `面试模式复习` 专注算法和面试题！」

#### 复习优先级规则

1. `category == "算法"` 的卡片优先于其他类别（面试临近时算法卡片权重 × 1.5）
2. 逾期天数多的优先
3. difficulty 低（记忆难度高）的优先

1. 读取 `data/flashcards.json`
2. 筛选 `nextReview <= today` 的卡片
3. 按上方优先级排序
4. 逐张展示：
   a. 显示正面（问题）
   b. 等用户回答（文字或说「翻面」）
   c. 显示背面（答案）
   d. 让用户自评：
      - 1️⃣ 完全忘了
      - 2️⃣ 勉强想起来
      - 3️⃣ 有点印象但不确定
      - 4️⃣ 记住了
      - 5️⃣ 非常熟悉
   e. 根据评分更新卡片参数
5. 复习完毕后报告：
   - 今日复习 X 张
   - 记住 / 模糊 / 忘记 的比例
   - 明天待复习数量

### 4. 每日推送（通过心跳/定时任务）

在 HEARTBEAT.md 或 cron 中配置：
- 每天 9:00 检查今日待复习卡片数
- 如果 > 0，通过钉钉推送提醒：「📚 今天有 X 张卡片待复习，加油！」
- 如果 = 0，不打扰

### 5. 统计报告

触发：用户说「复习统计」「记忆进度」

输出：
- 总卡片数 / 已掌握（interval > 21 天）/ 学习中 / 新卡片
- 本周复习次数 / 平均正确率
- 各类别掌握程度
- 最薄弱的 5 张卡片

### 6. 每周总结推送

触发：每周日（可在 HEARTBEAT.md 或 cron 中配置），或用户说「本周复习总结」

自动统计并通过钉钉推送：

```
📊 本周复习周报（MM-DD ~ MM-DD）

📚 本周复习卡片：X 张
✅ 掌握率：Y%（interval > 7 天的卡片占比）
🔥 连续打卡：Z 天

⚠️ 最薄弱的 3 张卡片（需要重点关注）：
1. [卡片正面] — 复习次数 N 次，当前 interval 仅 M 天
2. [卡片正面] — ...
3. [卡片正面] — ...

💡 建议：下周重点复习上述薄弱卡片，保持每日至少 10min 间隔复习！
```

统计逻辑：
- 本周复习数量：从 `data/review-log.json` 筛选本周日期的记录
- 掌握率：`interval > 7` 的卡片数 / 总卡片数
- 最薄弱卡片：`interval` 最短且 `repetitions` 最多（多次复习但仍记不住）的前 3 张

## FASTER 主动学习法（融入复习流程）

> 借鉴 learn-faster-kit 的 FASTER 方法论，让复习不只是"翻卡片"

**FASTER = Focus · Active Recall · Spaced Repetition · Testing · Elaboration · Review**

在复习时融入以下策略：

### F — Focus（聚焦）
- 每次复习前说出今天的目标类别（算法/JS/React/…）
- 超过 20 张待复习卡片时，优先处理逾期最久的前 15 张，其余顺延

### A — Active Recall（主动回忆）
- **不要立刻翻面**，先让用户尝试用自己的话回答
- 提示语："先不看答案，你能描述一下吗？"
- 用户回答后再对比标准答案，指出补充点

### S — Spaced Repetition（间隔重复）
- 即已有的 SM-2 算法，评分 < 3 重置 interval

### T — Testing（自测）
- 每 5 张卡片后插入一个"小测验"：随机抽一张之前复习过的卡，再测一次
- 触发词："测验模式"

### E — Elaboration（精细化）
- 对评分 3 分（模糊记住）的卡片，追问一个延伸问题：
  - "你能举一个实际项目中用到这个的例子吗？"
  - "这个知识点和 X 有什么区别？"
- 将用户补充的内容提示写回卡片 back 字段

### R — Review（回顾）
- 复习结束时生成 3 句话总结："今天复习了 X 类知识，最薄弱的是 Y，下次重点关注 Z"

## 交互风格

- 复习时简洁明了，不废话
- 答案展示要精炼，突出关键词
- **默认开启 Active Recall**：先让用户尝试回答，再展示答案
- 适时给出记忆技巧和关联知识
- 对模糊记住的卡片（评分3分）主动追问延伸问题
