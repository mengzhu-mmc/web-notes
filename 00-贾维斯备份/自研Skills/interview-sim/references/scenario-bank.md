# 场景题 & 系统设计题库

## 场景设计题（考察实际开发能力）

### 1. 无限滚动列表
**题目**：实现一个无限滚动列表，数据量可能达到 10 万条。
**考察点**：虚拟滚动 (只渲染可视区) → 滚动事件节流 → 动态高度处理 → Intersection Observer → 内存管理 → 骨架屏/占位
**追问**：动态高度怎么处理？图片懒加载？搜索/筛选功能？

### 2. 大文件上传
**题目**：实现一个支持断点续传的大文件上传功能。
**考察点**：File.slice 分片 → 并发上传 + 控制并发数 → MD5/SHA 计算秒传 → 断点续传（记录已上传分片）→ 进度条 → 失败重试
**追问**：如何实现秒传？暂停/恢复如何实现？Worker 计算 hash？

### 3. 实时协作编辑
**题目**：设计一个多人实时协作的文本编辑器。
**考察点**：WebSocket 双向通信 → OT 算法 / CRDT → 光标同步 → 冲突解决 → 离线编辑 → 版本历史
**追问**：如何处理两人同时编辑同一段？离线后重连如何合并？

### 4. 前端错误监控
**题目**：设计一个前端错误监控系统。
**考察点**：错误类型 (JS 运行时 / 资源加载 / Promise 未捕获 / 接口异常) → 采集方式 (window.onerror / addEventListener / unhandledrejection) → Source Map 还原 → 上报策略 (采样 / 批量 / 离线缓存) → 报警
**追问**：如何避免监控代码本身的错误？采样率怎么定？白屏检测？

### 5. 权限管理系统
**题目**：设计一个前端权限管理系统（按钮级别）。
**考察点**：RBAC 模型 → 路由权限 (动态路由 / 路由守卫) → 按钮权限 (指令/组件/函数) → 菜单权限 → 数据权限 → 权限缓存与更新
**追问**：角色继承怎么设计？权限变更实时生效？

### 6. 手写 debounce & throttle
**题目**：手写 debounce 和 throttle，支持 leading / trailing 选项。
**考察点**：基本实现 → leading/trailing 配置 → cancel 方法 → 返回值处理 → this 绑定 → 时间戳 vs 定时器

### 7. 手写并发控制器
**题目**：实现 `asyncPool(limit, tasks)`，控制最大并发数。
**考察点**：Promise 池 → race 检测完成 → 错误处理策略 → 取消支持 → 优先级队列

### 8. 图片懒加载
**题目**：实现一个高性能的图片懒加载方案。
**考察点**：Intersection Observer (替代 scroll 监听) → placeholder / 渐进加载 → srcset 响应式 → 预加载关键图片 → loading="lazy" 原生方案 → 骨架屏
**追问**：如何处理快速滚动？SSR 下如何处理？

### 9. 前端性能优化方案
**题目**：一个页面 FCP 3 秒，LCP 5 秒，如何优化到 1 秒？
**考察点**：诊断 (Lighthouse / Performance API / Chrome DevTools) → 资源优化 (压缩/CDN/Code Split) → 渲染优化 (SSR/预渲染/骨架屏) → 缓存策略 → 关键渲染路径 → 预加载 (prefetch/preload)
**追问**：具体量化指标？优先级排序？AB 测试？

### 10. 移动端适配
**题目**：如何做一个多端适配的 H5 页面？
**考察点**：viewport 设置 → rem/vw 方案 → 1px 问题 → 安全区域 (env(safe-area-inset-*)) → 响应式断点 → 图片适配 (@2x/@3x) → 键盘弹起适配

## 系统设计题（考察架构能力）

### S1. 设计一个组件库
**维度**：技术选型 (Monorepo / 构建工具) → 组件设计原则 (原子设计) → 主题定制 (CSS 变量 / Design Token) → 文档系统 → 测试策略 → 版本管理 → 按需引入 → A11y

### S2. 设计一个微前端架构
**维度**：方案选型 (qiankun / Module Federation / iframe) → 应用隔离 (JS/CSS 沙箱) → 通信方案 → 路由管理 → 共享依赖 → 部署策略 → 性能影响

### S3. 设计一个低代码平台
**维度**：编辑器架构 (画布/属性面板/组件树) → 组件协议 (Schema) → 拖拽实现 → 数据绑定 → 事件系统 → 预览/发布 → 版本管理 → 协作

### S4. 设计一个前端 CI/CD 流水线
**维度**：代码检查 (ESLint/TypeScript/单测) → 构建 (多环境配置) → 部署 (CDN/灰度/回滚) → 监控 (性能/错误/告警) → 通知 → 审批流

### S5. 设计一个 IM 聊天页面
**维度**：消息列表 (虚拟滚动/时间分组) → WebSocket 连接管理 (重连/心跳) → 消息类型 (文本/图片/文件/语音) → 已读未读 → 消息搜索 → 离线消息 → @提及 → 表情

### S6. 设计一个数据可视化大屏
**维度**：自适应布局 (缩放方案 scale vs rem) → 图表选型 (ECharts/D3/Canvas) → 实时数据更新 (轮询/WebSocket/SSE) → 性能 (大量数据点) → 联动交互 → 全屏展示
