## 美团 AI Agent 产品对比学习

靓仔让我读了两篇 KM 文档，系统了解美团 AI 产品生态。

### 五款产品一览

| 产品 | 定位 | 形态 | 目标用户 |
|---|---|---|---|
| **CatPaw** | AI 编程助手 & 研发效率平台 | CLI / IDE / 插件 / Agent SDK | 研发人员 |
| **CatPaw Desk** | 通用 Agent 桌面端（非编程场景） | 独立桌面 App (macOS/Win) | 研发 + 通用办公 |
| **小美搭档** | 通用 AI 办公助手 | 独立 App | 全公司员工 |
| **OpenClaw** | 自主 AI 智能体框架 | CLI + 多通道 + 原生 App | 有技术背景的用户 |
| **CatClaw** | OpenClaw 在大象上的落地形态 | 大象机器人 | 大象用户 |

### 关键区别

- **CatPaw**：研发全流程（编码/测试/部署），支持 MCP、Skills、Hooks、Browser Use
- **CatPaw Desk**：基于 Electron，内置浏览器录制→一键生成 Skill，面向非编程自动化
- **小美搭档**：开箱即用，模型黑盒不可切换，无需技术背景
- **OpenClaw**：7×24 自主运行，多模型路由，多通道接入，需本地部署
- **CatClaw**：纯云端，不依赖本地 OpenClaw（评论区闫亚军确认）

### CatPaw Cowork vs OpenClaw 架构对比

来源：km.sankuai.com/collabpage/2750511571

- CatPaw Cowork：Electron 三进程模型，MCP 标准协议，macOS Seatbelt 沙箱，嵌入式浏览器
- OpenClaw：Gateway + 多通道，自定义 Tool Schema + Skills，Docker 容器沙箱，独立浏览器进程

### 与靓仔讨论的技术概念

1. Agent vs MCP Host：Agent = MCP Host + 记忆系统 + 自主决策循环 + 多通道 + 多 Agent 协作
2. Agent 的循环机制：LLM 返回 tool_call → 继续执行，返回纯文本 → 任务结束
3. Agent 不内置 LLM，思考靠外部 LLM API 调用，记忆靠工程代码（文件/DB/向量存储）
4. MCP Host 如果实现了循环调度，核心工作流和 Agent 一样，Agent 只是在此基础上堆了更多工程能力

### OpenClaw 多 Agent 协作机制

来源：km.sankuai.com/collabpage/2749188704

#### 核心架构：Supervisor + Router 混合模式
- **主 Agent（Supervisor）**：接单、派单、质量把控、汇总结果
- **专家 Agent**：各自专注一个领域（如日报、小红书运营等）
- 每个 Agent 有独立的 workspace、model、身份标识

#### Agent 间通信：sessions_send
- 相当于"内线电话"
- 必须显式启用 `tools.agentToAgent.enabled: true`
- 白名单机制控制哪些 Agent 可以互相通信
- 支持结构化 JSON 通信减少理解偏差

#### 消息路由：Bindings
- 通过 `bindings` 配置，按 channel + accountId 精确匹配
- 不同机器人/通道 → 路由到不同 Agent
- 越具体的规则优先级越高

#### 三层物理隔离
1. **身份层**：每个 Agent 独立的 agent/ 目录（auth、models）
2. **状态层**：独立 sessions/ 目录
3. **工作层**：独立 workspace（SOUL.md、AGENTS.md、memory/）

#### 权限隔离
- 最小权限原则：每个 Agent 只开放必要的工具
- 例：reporter 禁用 browser（只需生成报告），xiaohongshu 需要 browser（自动化发布）

#### 部署步骤
1. `openclaw agents add <id> --model <model> --workspace <path>`
2. 编写灵魂文件（SOUL.md、AGENTS.md、USER.md）
3. `openclaw agents set-identity --agent <id> --name <name> --emoji <emoji>`
4. 配置 openclaw.json（bindings + agentToAgent + agents.list）
5. `openclaw gateway restart`
6. 验证路由和通信

#### 最佳实践
- 个人使用推荐 3-5 个 Agent（主管 + 2-4 专家）
- 两个 Agent 超过 80% 是同类任务 → 考虑合并
- 简单任务主管自己处理，复杂任务才派单
- 子 Agent 可用更轻量模型节省 Token

#### 预期效果
- 响应速度 +50%（独立记忆，无膨胀）
- 回答质量 +80%（专业分工，无上下文污染）
- Token 消耗 -40%（按需加载，无无关背景）

## 自建三个 Skill（14:51~15:05）

靓仔让我做自我能力审查并主动补强。分析后发现三个薄弱点，自建了三个 skill：

### 1. algo-coach（算法教练）
- 追踪力扣刷题进度，分析薄弱题型
- 按冲刺计划推荐每日题目
- 三级递进提示（不直接给答案）
- 解题复盘 + 进度统计
- 含 `references/topic-map.md`（120+ 道前端面试高频算法题分类）

### 2. interview-sim（前端面试模拟器）
- 模式 A：完整 45-60 分钟模拟面试（自我介绍→算法→八股→场景→反问）
- 模式 B：五个专项模块（算法/八股/场景/系统设计/行为面试）
- 四级评分 + 细分维度权重
- 含 `references/knowledge-bank.md`（30 道八股文题库）
- 含 `references/scenario-bank.md`（10 场景题 + 6 系统设计题）

### 3. spaced-review（间隔复习系统）
- SM-2 算法实现（Node.js 脚本 `scripts/sm2.js`，已测试通过）
- 从 notes/ 目录自动提取知识点生成闪卡
- 复习交互流程 + 自评打分
- 数据文件：`data/flashcards.json` + `data/review-log.json`（已初始化）
- 可配合心跳/cron 做每日推送

三个 skill 均通过 quick_validate 验证，已安装到 /app/skills/。
