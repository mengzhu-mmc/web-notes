## Webpack

- **构建流程**：入口 → 解析（Loader 转换）→ 生成 AST → 依赖图 → 输出（Plugin 介入各阶段）
- **Loader**：文件转换器（`css-loader` → `style-loader` → `babel-loader`），从右到左执行
- **Plugin**：扩展能力（`HtmlWebpackPlugin`、`MiniCssExtractPlugin`、`DefinePlugin`），监听生命周期事件
- **Tree Shaking**：依赖 ESM 静态分析（`import/export`），标记未使用代码，配合 Terser 清除
  - 局限：动态 `require()`、副作用代码（`sideEffects: false` 需谨慎）
- **Code Splitting**：动态 `import()` / `SplitChunksPlugin` / 路由级懒加载
- **HMR**：模块热替换，开发时不刷新页面更新模块（依赖 `module.hot`）

## Vite

- **为什么快**：开发时不打包，利用浏览器原生 ESM + esbuild 预构建依赖
- **依赖预构建**：把 CJS/UMD 转 ESM，合并小模块，减少请求数
- **生产构建**：用 Rollup（更好的 tree-shaking 和代码分割）
- **Plugin 生态**：兼容 Rollup 插件，扩展方式与 Webpack 不同

## Babel

- **三阶段**：Parse（生成 AST）→ Transform（插件处理）→ Generate（输出代码）
- `@babel/preset-env`：根据 targets 自动决定需要转换的语法 + polyfill
- **polyfill**：`@babel/polyfill`（已废弃）→ `core-js` + `regenerator-runtime`
- `useBuiltIns: 'usage'`：按需引入 polyfill（推荐）

## 模块化

| 特性 | CJS (CommonJS) | ESM (ES Modules) |
|---|---|---|
| 语法 | `require/exports` | `import/export` |
| 加载时机 | 运行时 | 编译时（静态分析） |
| 拷贝方式 | 值拷贝 | 值引用（live binding） |
| 循环依赖 | 可能拿到不完整导出 | 安全（静态分析可处理） |
| Tree Shaking | 不支持 | 支持 |

## 微前端

- **qiankun**：基于 single-spa，JS 沙箱 + CSS 隔离 + 应用生命周期
- **Module Federation**（Webpack 5）：模块级共享，运行时动态加载远程模块
- **核心问题**：依赖共享、样式隔离、路由同步、通信机制

## 包管理

- **npm**：树状 node_modules（可能重复）
- **yarn**：扁平化 + lock 文件
- **pnpm**：硬链接 + 符号链接，磁盘友好 + 严格依赖（避免幽灵依赖）

## CI/CD 与 Monorepo

- **Monorepo**：pnpm workspaces / Turborepo / Nx，统一版本管理 + 共享配置
- **CI/CD**：GitHub Actions / GitLab CI，自动化 lint → test → build → deploy
