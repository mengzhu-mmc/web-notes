## HTTP 版本

| 版本 | 关键特性 | 解决的问题 |
|---|---|---|
| HTTP/1.0 | 每次请求新建连接 | 基础 |
| HTTP/1.1 | 持久连接 + 管道化 | 减少握手开销 |
| HTTP/2 | 多路复用 + 头部压缩 + 服务端推送 | **队头阻塞**（同域名并发限制 6） |
| HTTP/3 | QUIC（基于 UDP）+ 0-RTT | **传输层队头阻塞**（TCP 丢包重传阻塞） |

## TCP

- **三次握手**：SYN → SYN+ACK → ACK（防止历史连接干扰，双方确认收发能力）
- **四次挥手**：FIN → ACK → FIN → ACK（**TIME_WAIT**：2MSL 等待，确保对方收到 ACK）
- **拥塞控制**：慢启动 → 拥塞避免 → 快重传 → 快恢复

## 缓存

```
请求 → 强缓存（Cache-Control/Expires）→ 命中则直接返回
     → 未命中 → 协商缓存（ETag/Last-Modified）→ 304 Not Modified
     → 协商未命中 → 200 返回新内容
```

- **强缓存**：`Cache-Control: max-age=31536000`（静态资源）/ `no-cache`（需协商）/ `no-store`（不缓存）
- **协商缓存**：`ETag`（内容哈希，精确）优先于 `Last-Modified`（时间戳，秒级精度）

## 跨域

- **同源策略**：协议 + 域名 + 端口 三者相同
- **CORS**：简单请求（GET/POST + 安全头部）直接发；预检请求（OPTIONS）先确认
- **其他方案**：Nginx 反向代理 / JSONP（仅 GET，已淘汰）/ `postMessage`

## 安全

| 攻击 | 原理 | 防御 |
|---|---|---|
| XSS | 注入恶意脚本 | 转义输出、CSP、`HttpOnly` Cookie |
| CSRF | 利用已登录身份发请求 | CSRF Token、`SameSite` Cookie、验证 Referer |
| 点击劫持 | iframe 覆盖诱导点击 | `X-Frame-Options` / CSP `frame-ancestors` |

- **CSP**（Content Security Policy）：限制资源加载来源，有效防御 XSS

## HTTPS/TLS

- **TLS 1.2**：4 次握手（客户端随机数 + 服务端随机数 → 生成会话密钥）
- **TLS 1.3**：1-RTT（减少握手延迟），废弃不安全加密套件

## WebSocket

- 基于 HTTP 升级（`Upgrade: websocket`），全双工通信
- 适合：实时聊天、推送通知、协同编辑

## DNS

- 递归查询 → 迭代查询 → 缓存（浏览器 → OS → 路由器 → ISP → 根域名）
- **优化**：DNS 预解析（`<link rel="dns-prefetch">`）、CDN 就近解析
