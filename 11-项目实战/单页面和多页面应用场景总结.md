### 单页面和多页面应用场景总结

## 一、SPA vs MPA 对比

| 维度 | SPA（单页面应用） | MPA（多页面应用） |
|------|-----------------|-----------------|
| **定义** | 只有一个 HTML 入口，页面切换通过 JS 动态替换 DOM | 每个页面独立 HTML 文件，页面跳转刷新整页 |
| **加载方式** | 首次加载所有资源，后续切换无需重新请求 HTML | 每次跳转都请求新的 HTML 文件 |
| **首屏速度** | 慢（资源包大） | 快（按需加载各页面资源） |
| **页面切换** | 流畅（无白屏，可加转场动画） | 有白屏闪烁 |
| **SEO** | 差（内容由 JS 渲染，爬虫难以抓取） | 好（服务端直接返回完整 HTML） |
| **开发体验** | 好（组件化、状态共享方便） | 相对分散，页面间通信复杂 |
| **代表框架** | Vue + Vue Router / React + React Router | 传统 PHP/JSP / Next.js（多页） |

---

## 二、什么是单页面应用（SPA）

*基本概念*：在项目应用中，以单个HTML页面作为外壳页面，并在外壳页面一次性加载项目所依赖的资源。项目中其他页面作为页面片段在外壳页面中进行切换（例如：利用H5的History监听URL的变化，对页面片段进行切换）

*单页面应用结构图：*

<img src="./单页面结构图.png" />

**优点：** 项目依赖资源公用，页面片段切换流畅，页面片段可以复用，页面片段切换的时候可以使用转场动画

**缺点：** 不利于SEO优化，首屏加载缓慢

***解决首屏加载慢的方案：***

1. 减小入口文件的体积：常用手段是路由懒加载，只有在解析路由时才加载组件

2. 静态资源缓存：

    * 采用HTTP缓存后端返回的某些资源

    * CDN静态资源缓存：react、react-dom、axios

3. 按需引用UI框架：比如：

    ```js
    import { Button } from 'antd'
    ```

4. 图片资源压缩

---

## 三、什么是多页面应用（MPA）

这种技术方案就是最初搭建页面的实现方式，每个页面都有各自的HTML文件，每个页面所依赖的资源都在各自HTML文件中引入，并且页面之间的跳转通过URL跳转。

---

## 四、如何选择页面结构

具体根据项目分析：

- **官网、电商类**：对SEO和首屏加载速度要求较高 → 采用多页面应用或服务端渲染（Next.js / Nuxt.js）
- **后台管理系统**：不对外开放，不需要SEO → 单页面应用，利用 Vue/React 组件化开发；系统过大时可拆分为多个子应用（微前端）
- **登录页**：通常作为后台管理系统的独立入口页

## 五、代码示例

### Vue Router：SPA 路由配置（含懒加载）

```javascript
// router/index.js - Vue 3 + Vue Router 4
import { createRouter, createWebHistory } from 'vue-router';

const router = createRouter({
  history: createWebHistory(), // History 模式（需服务端配置 fallback）
  // history: createWebHashHistory(), // Hash 模式（URL 含 #，无需服务端配置）
  routes: [
    {
      path: '/',
      component: () => import('@/views/Home.vue'),  // 路由懒加载（代码分割）
    },
    {
      path: '/dashboard',
      // 带 chunk 名，便于分析产物
      component: () => import(/* webpackChunkName: "dashboard" */ '@/views/Dashboard.vue'),
      meta: { requiresAuth: true },
    },
    {
      path: '/login',
      component: () => import('@/views/Login.vue'),
    },
    { path: '/:pathMatch(.*)*', component: () => import('@/views/404.vue') },
  ],
});

// 路由守卫：鉴权
router.beforeEach((to, from) => {
  if (to.meta.requiresAuth && !isLoggedIn()) {
    return { path: '/login', query: { redirect: to.fullPath } };
  }
});

export default router;
```

### React Router：SPA 路由配置（含懒加载）

```jsx
// App.jsx - React 18 + React Router v6
import React, { Suspense, lazy } from 'react';
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom';

// 懒加载路由组件
const Home = lazy(() => import('./pages/Home'));
const Dashboard = lazy(() => import('./pages/Dashboard'));
const Login = lazy(() => import('./pages/Login'));

function PrivateRoute({ children }) {
  return isLoggedIn() ? children : <Navigate to="/login" replace />;
}

export default function App() {
  return (
    <BrowserRouter>
      <Suspense fallback={<div>Loading...</div>}>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/login" element={<Login />} />
          <Route
            path="/dashboard"
            element={<PrivateRoute><Dashboard /></PrivateRoute>}
          />
        </Routes>
      </Suspense>
    </BrowserRouter>
  );
}
```

### History 模式 vs Hash 模式对比

```
Hash 模式（createWebHashHistory）
  URL：http://example.com/#/dashboard
  ✅ 无需服务端配置（# 后内容不发送到服务端）
  ❌ URL 不美观，SEO 不友好

History 模式（createWebHistory）
  URL：http://example.com/dashboard
  ✅ URL 美观，SEO 友好
  ❌ 需要服务端配置：所有路径都返回 index.html（否则刷新 404）

  Nginx 配置示例：
  location / {
    try_files $uri $uri/ /index.html;
  }
```
