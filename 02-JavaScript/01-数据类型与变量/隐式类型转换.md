# 隐式类型转换

> 面试高频考点 | 搞懂 JS 的"暗箱操作"

## 面试高频考点

- `==` 和 `===` 的区别？
- `[] == ![]` 为什么是 `true`？
- ToPrimitive 的转换规则是什么？
- 如何让 `a == 1 && a == 2 && a == 3` 成立？

---

## 一、三种基础转换操作

| 转换 | 规则 |
|------|------|
| `ToPrimitive(obj)` | 对象转原始值：默认先调 `valueOf`，再调 `toString`；Date 类型优先 `toString` |
| `ToNumber(x)` | `undefined → NaN`、`null → 0`、`true → 1`、`false → 0`、字符串按内容转、对象先 ToPrimitive |
| `ToBoolean(x)` | `0、NaN、''、null、undefined、false` → `false`；其余（包括 `[]`、`{}`）→ `true` |

### ToPrimitive 详解

```js
// 优先级：Symbol.toPrimitive > valueOf > toString
const obj = {
  [Symbol.toPrimitive]() { return 6; },
  valueOf() { return 4; },
  toString() { return '5'; }
};
obj + 1; // 7（Symbol.toPrimitive 优先）

// 普通对象
({}).valueOf();  // 返回 this（不是原始值），继续调 toString
({}).toString(); // "[object Object]"

// 数组
[].valueOf();    // 返回 this（不是原始值），继续调 toString
[].toString();   // ""

// Date 优先 toString
new Date().valueOf();  // 时间戳（毫秒数，number）
new Date().toString(); // "Thu Mar 26 2026..."
// 隐式转换时 Date 优先用 toString
```

## 二、`==` 转换规则（精简版）

```
1. 类型相同 → 直接比较（NaN !== NaN 是特例）
2. null == undefined → true（唯一特例，不再转换）
3. 一方是 Boolean → 先把 Boolean 转为 Number，再重新比较
4. 一方是 Number，另一方是 String → String 转 Number
5. 一方是 Object，另一方是原始类型 → Object 做 ToPrimitive，再重新比较
6. 其他情况 → false
```

### 经典题目解析

```js
// 1. [] == ![]  → true
// 步骤：
// ![] → false（[] 是对象，ToBoolean 为 true，取反得 false）
// [] == false → false 先转 Number：false → 0 → [] == 0
// [] == 0 → [] 做 ToPrimitive：[].toString() = '' → '' == 0
// '' == 0 → '' 转 Number：0 → 0 == 0 → true ✅

// 2. {} + [] → 0（在命令行里）
// {} 被解析为空代码块，+[] → +([] → '') → +'' → 0

// 3. [] + {} → "[object Object]"
// [] → ''，{} → "[object Object]"，字符串拼接

// 4. null == 0  → false（null 只等于 undefined，不转换）
// 5. null == false → false（同上）
// 6. undefined == false → false
// 7. NaN == NaN → false（NaN 不等于任何值包括自身）

// 8. 如何让 a == 1 && a == 2 && a == 3 成立？
const a = {
  i: 1,
  valueOf() {
    return this.i++;
  }
};
a == 1 && a == 2 && a == 3; // true
// 原理：a == number → ToPrimitive(a) → 调 valueOf → 返回 i 并自增
```

## 三、加法运算符 `+` 的特殊规则

`+` 是唯一一个"有字符串就字符串拼接"的运算符。

```js
// 规则：两个操作数先 ToPrimitive，有字符串就拼接，否则 ToNumber 相加
'1' + 2       // '12'（字符串拼接）
1 + true      // 2（Number + Number）
1 + null      // 1（null → 0）
1 + undefined // NaN（undefined → NaN）
1 + []        // '1'（[] → '' → 字符串拼接）
1 + {}        // '1[object Object]'（{} → '[object Object]' → 字符串拼接）
[] + []       // ''
[] + {}       // '[object Object]'
{} + []       // 0（{} 解析为代码块，+[] → 0）

// 减乘除：全部强制 ToNumber
'3' - 1       // 2
'3' * '2'     // 6
[] - 1        // -1（[] → '' → 0，0-1）
null - 1      // -1（null → 0）
undefined - 1 // NaN
```

## 四、ToBoolean 转换（if / && / || / ! 触发）

```js
// 假值（6 个）：转为 false
Boolean(false)     // false
Boolean(0)         // false（包括 -0、+0）
Boolean(NaN)       // false
Boolean('')        // false（空字符串）
Boolean(null)      // false
Boolean(undefined) // false

// 真值：其他所有值（包括空数组、空对象）
Boolean([])    // true ← 面试陷阱！
Boolean({})    // true ← 面试陷阱！
Boolean('0')   // true（非空字符串）
Boolean(-1)    // true（非零数字）

// 短路运算
0 || 'default'     // 'default'（0 是假值）
'' || 'fallback'   // 'fallback'
null ?? 'default'  // 'default'（?? 只认 null/undefined）
0 ?? 'default'     // 0（0 不是 null/undefined，?? 不触发）
```

## 五、`+` 一元运算符 vs `!!` 双重取反

```js
// 一元 + 强制转 Number
+'3'        // 3
+true       // 1
+false      // 0
+null       // 0
+undefined  // NaN
+[]         // 0（[] → '' → 0）
+{}         // NaN（{} → '[object Object]' → NaN）

// !! 强制转 Boolean（比 Boolean() 更简洁）
!!0         // false
!!''        // false
!!null      // false
!![]        // true
!!{}        // true
```

## 六、总结：面试快速回忆框架

```
== 比较遇到 Boolean → 先把 Boolean 转 Number
== 比较遇到 Object  → 先把 Object 做 ToPrimitive
null / undefined 只互等，不转换

+ 运算符：有字符串就拼，否则转 Number
- * / 运算符：全部转 Number

if / || / && / ! ：ToBoolean（假值只有 6 个）
```
