# 模块化详解

> CommonJS vs ES Module 核心区别与实战

---

## CommonJS（Node.js 默认）

```js
// 导出
module.exports = { foo, bar };
exports.foo = foo;

// 导入
const { foo, bar } = require('./module');
```

**特点：**
- **运行时加载**：执行到 require 时才加载
- **值拷贝**：导出的是值的拷贝，模块内部变化不影响已导入的值
- **同步加载**：不适合浏览器环境
- **动态导入**：`require(variable)` 支持运行时决定

## ES Module（ES2015 标准）

```js
// 导出
export const foo = 1;
export function bar() {}
export default MyClass;

// 导入
import { foo, bar } from './module';
import MyClass from './module';
import * as mod from './module';
```

**特点：**
- **编译时加载**：静态分析，构建时可 tree-shaking
- **引用（Live Binding）**：导入的是引用，模块内部变化会反映到导入方
- **异步加载**：适合浏览器
- **严格模式**：模块内自动启用 strict mode

## 核心区别

| 特性 | CommonJS | ES Module |
|---|---|---|
| 加载时机 | 运行时 | 编译时（静态） |
| 导出方式 | module.exports | export / export default |
| 导入方式 | require() | import |
| 值的传递 | 值拷贝 | 引用（Live Binding） |
| this 指向 | 指向 module | undefined |
| 动态导入 | 天然支持 | `import()` 函数 |
| Tree Shaking | 不支持 | 支持 |
| 顶层 await | 不支持 | 支持 |

## 循环依赖

```js
// a.js
import { b } from './b';
export const a = 'a';

// b.js
import { a } from './a';  // a 可能还未初始化
export const b = 'b';
```

**ESM 循环依赖**：因为是引用，可能拿到 `undefined`。解决：延迟访问（函数内引用）。

**CJS 循环依赖**：拿到的是当前已执行的部分导出。

## 动态导入

```js
// 返回 Promise，支持运行时决定
const module = await import('./module');

// 按需加载（代码分割）
button.onclick = async () => {
  const { heavyFunction } = await import('./heavy-module');
  heavyFunction();
};
```

## Node.js 中使用 ESM

```json
// package.json
{ "type": "module" }
```

或使用 `.mjs` 扩展名。

## 最佳实践

1. **浏览器项目**：优先使用 ES Module（支持 tree-shaking）
2. **Node.js 项目**：ESM 是趋势，但 CJS 生态仍广泛
3. **库开发**：同时提供 ESM 和 CJS 版本
4. **避免混用**：同一文件不要混用 require 和 import
