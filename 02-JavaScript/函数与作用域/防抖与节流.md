# 防抖与节流

## 面试高频考点

- 防抖和节流的区别是什么？
- 分别在什么场景下使用？
- 如何手写防抖和节流？

---

## 一、核心区别

| 对比项 | 防抖（Debounce） | 节流（Throttle） |
| --- | --- | --- |
| 触发时机 | 停止触发后延迟执行 | 固定时间间隔内只执行一次 |
| 典型场景 | 搜索框输入、窗口 resize | 滚动监听、按钮防重复点击 |
| 核心思路 | 重置计时器 | 记录上次执行时间 |

**一句话记忆**：防抖是"等你停下来再说"，节流是"说了你也得等一会儿"。

---

## 二、手写防抖

```js
/**
 * 防抖：在 wait 毫秒内多次触发，只执行最后一次
 * @param {Function} fn - 要防抖的函数
 * @param {number} wait - 等待时间（ms）
 * @param {boolean} immediate - 是否立即执行（首次触发立即执行）
 */
function debounce(fn, wait, immediate = false) {
  let timer = null;

  return function (...args) {
    const context = this;

    // 如果设置了立即执行，且当前没有定时器（首次触发）
    if (immediate && !timer) {
      fn.apply(context, args);
    }

    // 清除上一个定时器，重新计时
    clearTimeout(timer);

    timer = setTimeout(() => {
      timer = null;
      if (!immediate) {
        fn.apply(context, args);
      }
    }, wait);
  };
}

// 使用示例
const handleSearch = debounce((value) => {
  console.log('搜索：', value);
}, 500);

input.addEventListener('input', (e) => handleSearch(e.target.value));
```

## 三、手写节流

### 方式一：时间戳版（首次立即执行）

```js
function throttle(fn, wait) {
  let lastTime = 0;

  return function (...args) {
    const now = Date.now();

    if (now - lastTime >= wait) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}
```

### 方式二：定时器版（最后一次也会执行）

```js
function throttle(fn, wait) {
  let timer = null;

  return function (...args) {
    if (!timer) {
      timer = setTimeout(() => {
        timer = null;
        fn.apply(this, args);
      }, wait);
    }
  };
}
```

### 方式三：结合版（首次立即 + 最后一次也执行）

```js
function throttle(fn, wait) {
  let lastTime = 0;
  let timer = null;

  return function (...args) {
    const now = Date.now();
    const remaining = wait - (now - lastTime); // 距离下次执行还剩多少时间

    if (remaining <= 0) {
      // 已经超过间隔，立即执行
      if (timer) {
        clearTimeout(timer);
        timer = null;
      }
      lastTime = now;
      fn.apply(this, args);
    } else {
      // 还没到时间，设置定时器保证最后一次触发也能执行
      clearTimeout(timer);
      timer = setTimeout(() => {
        lastTime = Date.now();
        timer = null;
        fn.apply(this, args);
      }, remaining);
    }
  };
}
```

## 四、实际应用场景

### 防抖适用场景

```js
// 1. 搜索框输入联想
searchInput.addEventListener('input', debounce(fetchSuggestions, 300));

// 2. 窗口 resize 后重新计算布局
window.addEventListener('resize', debounce(recalculateLayout, 200));

// 3. 表单验证（用户停止输入后再验证）
emailInput.addEventListener('input', debounce(validateEmail, 500));
```

### 节流适用场景

```js
// 1. 滚动加载更多
window.addEventListener('scroll', throttle(loadMore, 200));

// 2. 鼠标移动事件（如拖拽）
element.addEventListener('mousemove', throttle(handleDrag, 16)); // ~60fps

// 3. 按钮防重复点击
submitBtn.addEventListener('click', throttle(submitForm, 1000));
```

## 五、面试追问

**Q：防抖和节流哪个性能更好？**

取决于场景。防抖在高频触发时只执行一次，节流会按固定频率执行。如果只关心最终结果用防抖，如果需要过程中持续响应用节流。

**Q：React 中如何使用防抖/节流？**

```jsx
import { useCallback, useRef } from 'react';

// 方式一：useMemo/useCallback 包裹（注意：每次渲染会重新创建）
const debouncedSearch = useCallback(debounce(fetchData, 300), []);

// 方式二：useRef 保存（推荐，避免重新创建）
const debouncedRef = useRef(debounce(fetchData, 300));
const handleInput = (e) => debouncedRef.current(e.target.value);

// 方式三：使用 ahooks 的 useDebounceFn / useThrottleFn
import { useDebounceFn } from 'ahooks';
const { run } = useDebounceFn(fetchData, { wait: 300 });
```
