## 一、什么是闭包

**MDN 定义**：闭包是指那些能够访问自由变量的函数（自由变量：既不是函数参数，也不是函数局部变量，而是另一个函数作用域中的变量）。

**红宝书定义**：闭包是指有权访问另外一个函数作用域中变量的函数。

**一句话理解**：函数 + 其词法环境（能访问的外部变量）= 闭包。

---

## 二、闭包产生的原因

核心：**当前环境中存在指向父级作用域的引用**。

```js
function f1() {
  var a = 2;
  function f2() {
    console.log(a); // 2 —— f2 引用了 f1 的作用域变量
  }
  return f2;
}

var x = f1();
x(); // 2 —— f1 执行完毕，但 a 没有被回收，因为 f2 还持有引用
```

作用域链：f2 的作用域链 → f1 的作用域 → 全局作用域 → null

## 三、闭包的表现形式

### 1. 返回函数（最常见）

```js
function makeCounter() {
  let count = 0;
  return {
    increment() { count++; },
    decrement() { count--; },
    getCount() { return count; }
  };
}
const counter = makeCounter();
counter.increment();
counter.getCount(); // 1
```

### 2. 作为函数参数传递

```js
var a = 1;
function foo() {
  var a = 2;
  function baz() {
    console.log(a); // 2，不是 1
  }
  bar(baz);
}
function bar(fn) {
  fn(); // 闭包：fn 携带了 foo 的作用域
}
foo();
```

### 3. 异步回调（定时器、事件监听、Ajax）

```js
function delayLog(msg) {
  setTimeout(function () {
    console.log(msg); // 闭包：回调函数访问了外部的 msg
  }, 1000);
}
delayLog('hello'); // 1秒后输出 hello
```

### 4. IIFE（立即执行函数）

```js
var a = 2;
(function IIFE() {
  console.log(a); // 2 —— 访问了全局作用域的 a
})();
```

### 5. 模块模式（封装私有变量）

```js
const module = (function () {
  let _private = 0; // 私有变量，外部无法直接访问

  return {
    get() { return _private; },
    set(val) { _private = val; }
  };
})();

module.set(10);
module.get(); // 10
// module._private // undefined，无法访问
```

## 四、经典面试题：for 循环 + setTimeout

```js
// 问题：输出什么？
for (var i = 1; i <= 5; i++) {
  setTimeout(function timer() {
    console.log(i);
  }, 0);
}
// 输出：6 6 6 6 6
// 原因：setTimeout 是宏任务，循环结束后才执行，此时 i 已经是 6
```

**解决方案一：IIFE 创建独立作用域**

```js
for (var i = 1; i <= 5; i++) {
  (function (j) {
    setTimeout(function timer() {
      console.log(j); // 1 2 3 4 5
    }, 0);
  })(i);
}
```

**解决方案二：setTimeout 第三个参数**

```js
for (var i = 1; i <= 5; i++) {
  setTimeout(function timer(j) {
    console.log(j); // 1 2 3 4 5
  }, 0, i);
}
```

**解决方案三：let 块级作用域（推荐）**

```js
for (let i = 1; i <= 5; i++) {
  setTimeout(function timer() {
    console.log(i); // 1 2 3 4 5
  }, 0);
}
// let 每次循环都创建新的块级作用域，每个 i 都是独立的
```

## 五、闭包的优缺点

### 优点

- **封装私有变量**：模拟私有属性，避免全局污染
- **保持状态**：函数执行完毕后，变量不被回收，可以持久保存状态
- **模块化**：实现模块模式，隐藏实现细节

### 缺点

- **内存泄漏风险**：闭包持有外部变量引用，导致变量无法被垃圾回收
- **性能开销**：作用域链查找比局部变量慢

### 如何避免内存泄漏

```js
function createClosure() {
  const largeData = new Array(1000000).fill('data');
  return function () {
    console.log(largeData.length);
  };
}

let fn = createClosure();
fn();
fn = null; // ✅ 手动解除引用，让 GC 回收 largeData
```

## 六、闭包与 this 的陷阱

```js
const obj = {
  name: 'obj',
  getName: function () {
    // 普通函数：this 指向调用者
    return function () {
      console.log(this.name); // undefined（严格模式）或 window.name
    };
  }
};

// 解决方案一：保存 this
const obj2 = {
  name: 'obj2',
  getName: function () {
    const self = this; // 保存 this
    return function () {
      console.log(self.name); // 'obj2'
    };
  }
};

// 解决方案二：箭头函数（推荐）
const obj3 = {
  name: 'obj3',
  getName: function () {
    return () => {
      console.log(this.name); // 'obj3'，箭头函数继承外层 this
    };
  }
};
```
