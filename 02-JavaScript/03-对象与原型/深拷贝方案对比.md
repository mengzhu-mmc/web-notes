# 深拷贝方案对比

## 一、四种主流方案

### 1. structuredClone（原生 API，现代首选）

`structuredClone` 是浏览器和 Node.js 内置的深拷贝 API（Chrome 98+、Node 17+、Safari 15.4+），基于 [结构化克隆算法](https://developer.mozilla.org/zh-CN/docs/Web/API/Web_Workers_API/Structured_clone_algorithm)。

**支持的类型**：`Date`、`RegExp`、`Map`、`Set`、`ArrayBuffer`、`Blob`、`Error`、`TypedArray`，**原生支持循环引用**。

**不支持的类型**（抛出 `DataCloneError`）：函数、DOM 节点、Symbol。

```js
const original = {
  date: new Date(),
  map: new Map([['key', 1]]),
  set: new Set([1, 2, 3]),
  regex: /hello/gi,
  nested: { arr: [1, [2, 3]] }
}

const clone = structuredClone(original)
clone.date instanceof Date   // ✅ true
clone.map instanceof Map     // ✅ true
clone.nested.arr !== original.nested.arr // ✅ true，深拷贝

// 支持循环引用
const obj = { a: 1 }
obj.self = obj
const clonedCircular = structuredClone(obj) // ✅ 不报错

// ❌ 不支持函数
structuredClone({ fn: () => {} }) // DataCloneError
```

### 2. lodash.cloneDeep

兼容性好，支持旧环境（IE11+）。对函数会直接引用（浅拷贝），Symbol 会被忽略。`Map`/`Set` 可能被降级处理。

### 3. JSON.parse(JSON.stringify())

最简单，但局限性最大。

**失真的类型**：`Date` → 字符串、`undefined` → 丢失、函数 → 丢失、`NaN`/`Infinity` → `null`、`Map`/`Set` → `{}`、不支持循环引用（报错）。

### 4. 手写递归（最可控）

可以精确控制每种类型的处理，见下文。

## 二、性能对比

> 测试环境：Node 20，100 次取平均（数据仅供参考，实际受数据结构影响大）

| 方案 | 小对象（~10 属性） | 中对象（~100 属性） | 大对象（~1000 属性） | 备注 |
|------|-----------------|------------------|------------------|------|
| `structuredClone` | ⚡ 快 | ⚡ 快 | ⚡ 最快 | C++ 原生实现 |
| `JSON.parse/stringify` | 🟡 中 | 🟡 中 | 🟡 中 | 序列化开销 |
| `lodash.cloneDeep` | 🐢 慢 | 🐢 慢 | 🐢 最慢 | JS 实现，功能全 |
| 手写递归（简单版） | ⚡ 快 | 🟡 中 | 🟡 中 | 取决于实现质量 |

**结论**：数据量大、不含函数 → 优先 `structuredClone`；需要处理函数/兼容旧浏览器 → `lodash.cloneDeep`。

## 四、深拷贝手写实现（含循环引用）

```js
function deepClone(target, cache = new Map()) {
  // 基本类型直接返回
  if (typeof target !== 'object' || target === null) return target;
  // 循环引用检查
  if (cache.has(target)) return cache.get(target);
  // 特殊类型
  if (target instanceof Date) return new Date(target);
  if (target instanceof RegExp) return new RegExp(target.source, target.flags);
  // 初始化（先存再递归，处理循环引用）
  const cloneResult = Array.isArray(target) ? [] : {};
  cache.set(target, cloneResult);
  for (let key in target) {
    if (target.hasOwnProperty(key)) {
      cloneResult[key] = deepClone(target[key], cache);
    }
  }
  return cloneResult;
}
```

## 五、使用建议

```javascript
// 现代项目推荐 structuredClone
const clone1 = structuredClone(data);

// 需要兼容旧环境或处理函数时用 lodash
import cloneDeep from 'lodash/cloneDeep';
const clone2 = cloneDeep(data);

// 兼容方案
const myClone = (obj) => {
  if (typeof structuredClone !== 'undefined') {
    try { return structuredClone(obj); }
    catch { return cloneDeep(obj); }
  }
  return cloneDeep(obj);
};
```

## 六、对象扁平化（flatten）

将嵌套对象拍平为点分路径形式。

```js
function flatten(obj, prefix = '') {
  const result = {};
  for (const key in obj) {
    if (!obj.hasOwnProperty(key)) continue;
    const newKey = prefix ? `${prefix}.${key}` : key;
    if (typeof obj[key] === 'object' && obj[key] !== null && !Array.isArray(obj[key])) {
      Object.assign(result, flatten(obj[key], newKey));
    } else {
      result[newKey] = obj[key];
    }
  }
  return result;
}

// 示例
flatten({ a: { b: { c: 1 }, d: 2 }, e: 3 });
// → { 'a.b.c': 1, 'a.d': 2, 'e': 3 }
```

## 七、数组转树形结构

### 递归版（简洁但 O(n²)）

```js
function arrayToTree(arr, parentId = null) {
  return arr
    .filter(item => item.parentId === parentId)
    .map(item => ({
      ...item,
      children: arrayToTree(arr, item.id)
    }));
}
```

### Map 优化版（O(n)，面试推荐）

```js
function arrayToTree(arr) {
  const map = new Map();
  const roots = [];
  // 第一遍：建索引
  arr.forEach(item => map.set(item.id, { ...item, children: [] }));
  // 第二遍：挂载
  arr.forEach(item => {
    const node = map.get(item.id);
    if (item.parentId === null) roots.push(node);
    else map.get(item.parentId)?.children.push(node);
  });
  return roots;
}

// 示例数据
const arr = [
  { id: 1, parentId: null, name: '根' },
  { id: 2, parentId: 1, name: '子1' },
  { id: 3, parentId: 1, name: '子2' },
  { id: 4, parentId: 2, name: '孙1' },
];
// arrayToTree(arr) → [{ id:1, children: [{id:2, children:[{id:4,...}]}, {id:3,...}] }]
```

**核心优化点**：用 Map 存 id → node 的映射，把两层嵌套循环降为两次线性遍历，时间复杂度从 O(n²) 降为 O(n)。
