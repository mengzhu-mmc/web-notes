# 工厂模式 Factory Pattern

> 用一个函数（或类）来创建对象，隐藏创建细节

---

## 简单工厂

```js
function createPerson(type, name) {
  switch (type) {
    case 'developer':
      return { name, role: '开发', skills: ['JS', 'CSS'] };
    case 'designer':
      return { name, role: '设计', skills: ['Figma', 'Sketch'] };
    case 'pm':
      return { name, role: '产品', skills: ['需求分析', '原型'] };
    default:
      throw new Error('未知类型');
  }
}

const dev = createPerson('developer', 'Tom');
```

## 工厂方法

```js
class PersonFactory {
  static createDeveloper(name) {
    return new Developer(name);
  }
  static createDesigner(name) {
    return new Designer(name);
  }
}

class Developer {
  constructor(name) { this.name = name; this.role = '开发'; }
  code() { console.log(`${this.name} 写代码`); }
}

class Designer {
  constructor(name) { this.name = name; this.role = '设计'; }
  draw() { console.log(`${this.name} 画图`); }
}
```

## 前端常见场景

### 1. 组件工厂
```js
function createComponent(type, props) {
  const components = {
    Button: ButtonComponent,
    Input: InputComponent,
    Modal: ModalComponent
  };
  const Comp = components[type];
  if (!Comp) throw new Error(`未知组件: ${type}`);
  return new Comp(props);
}
```

### 2. Axios 请求工厂
```js
function createRequest(config) {
  const instance = axios.create({
    baseURL: config.baseURL,
    timeout: config.timeout || 10000,
    headers: config.headers || {}
  });

  // 统一拦截器
  instance.interceptors.response.use(
    res => res.data,
    err => Promise.reject(handleError(err))
  );

  return instance;
}
```

### 3. DOM 元素工厂
```js
function createElement(tag, attrs = {}, children = []) {
  const el = document.createElement(tag);
  Object.entries(attrs).forEach(([k, v]) => el.setAttribute(k, v));
  children.forEach(child => {
    el.appendChild(typeof child === 'string' ? document.createTextNode(child) : child);
  });
  return el;
}
```

## 优点
- **解耦**：调用方不需要知道具体类的实现
- **扩展**：新增产品类型只需添加新分支/新类
- **统一**：集中管理对象创建逻辑

## 缺点
- 违反开闭原则（简单工厂新增类型需修改 switch）
- 工厂方法会引入大量子类
