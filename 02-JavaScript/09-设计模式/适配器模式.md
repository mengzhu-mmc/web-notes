# 适配器模式 Adapter Pattern

> 将一个接口转换为另一个期望的接口，解决兼容问题

---

## 核心思想

```js
// 旧接口
function oldAPI() {
  return { data: [1, 2, 3], status: 'ok' };
}

// 新接口期望格式
function newAPI() {
  return { result: [1, 2, 3], code: 200 };
}

// 适配器
function apiAdapter(fn) {
  return function(...args) {
    const res = fn(...args);
    return {
      result: res.data,
      code: res.status === 'ok' ? 200 : 500
    };
  };
}

const adaptedAPI = apiAdapter(oldAPI);
```

## 前端常见场景

### 1. 第三方库适配
```js
// Axios 适配 fetch 风格
function fetchAdapter(axiosInstance) {
  return async function(url, options = {}) {
    const res = await axiosInstance({
      url,
      method: options.method || 'GET',
      headers: options.headers,
      data: options.body
    });
    return {
      ok: res.status >= 200 && res.status < 300,
      status: res.status,
      json: () => Promise.resolve(res.data)
    };
  };
}
```

### 2. 数据格式适配
```js
// 后端返回 → 前端组件需要的格式
function adaptUserData(raw) {
  return {
    fullName: `${raw.first_name} ${raw.last_name}`,
    avatar: raw.avatar_url || '/default.png',
    joinDate: new Date(raw.created_at).toLocaleDateString(),
    isAdmin: raw.role === 'admin'
  };
}
```

### 3. 事件适配
```js
// 将 jQuery 事件适配为原生事件风格
function jQueryEventAdapter($el) {
  return {
    on(event, handler) {
      $el.on(event, (e) => handler(e.originalEvent));
    },
    off(event, handler) {
      $el.off(event, handler);
    }
  };
}
```

### 4. 浏览器兼容
```js
// requestAnimationFrame 兼容
const requestAnimFrame = (function() {
  return window.requestAnimationFrame ||
    window.webkitRequestAnimationFrame ||
    window.mozRequestAnimationFrame ||
    function(callback) { window.setTimeout(callback, 1000 / 60); };
})();
```

## 优点
- 不修改原有代码，通过适配器桥接
- 适合旧系统迁移、第三方库集成

## 与装饰器的区别
- **适配器**：改变接口形状
- **装饰器**：不改变接口，只增强功能
