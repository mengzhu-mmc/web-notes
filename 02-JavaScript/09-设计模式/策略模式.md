# 策略模式 Strategy Pattern

> 将算法/策略封装为独立模块，运行时可自由切换

---

## 核心思想

```js
// 策略对象
const strategies = {
  vip(user, price) { return price * 0.8; },
  normal(user, price) { return price; },
  new(user, price) { return price - 10; }
};

// 上下文
function calculatePrice(userType, price) {
  const strategy = strategies[userType];
  if (!strategy) throw new Error('未知策略');
  return strategy(null, price);
}

calculatePrice('vip', 100);   // 80
calculatePrice('normal', 100); // 100
```

## 前端常见场景

### 1. 表单校验
```js
const validators = {
  required: (val) => val ? '' : '必填项',
  minLength: (val, len) => val.length >= len ? '' : `最少${len}个字符`,
  isEmail: (val) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(val) ? '' : '邮箱格式错误',
  isPhone: (val) => /^1[3-9]\d{9}$/.test(val) ? '' : '手机号格式错误'
};

function validate(field, rules) {
  for (const { strategy, message } of rules) {
    const [name, ...params] = strategy.split(':');
    const result = validators[name](field, ...params);
    if (result) return message || result;
  }
  return '';
}

// 使用
validate('test', [
  { strategy: 'required', message: '不能为空' },
  { strategy: 'isEmail', message: '邮箱格式不对' }
]);
```

### 3. 排序策略
```js
const sortStrategies = {
  byName: (a, b) => a.name.localeCompare(b.name),
  byPrice: (a, b) => a.price - b.price,
  byDate: (a, b) => new Date(b.date) - new Date(a.date)
```

## 优点
- **开闭原则**：新增策略无需修改原有代码
- **消除 if-else**：用策略映射替代冗长分支
- **可复用**：策略可独立测试和复用

## 适用场景
- 同一行为有多种实现
- 需要运行时切换算法
- 大量 if-else / switch 分支
