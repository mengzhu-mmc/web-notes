# 单例模式 Singleton Pattern

> 保证一个类只有一个实例，并提供全局访问点

---

## 核心实现

### 1. 闭包实现（经典）
```js
const Singleton = (function() {
  let instance = null;

  return function(name) {
    if (!instance) {
      instance = { name, getName: () => name };
    }
    return instance;
  };
})();

const a = new Singleton('aaa');
const b = new Singleton('bbb');
console.log(a === b); // true — 只实例化一次
```

### 2. ES6 Class 实现
```js
class Singleton {
  constructor(name) {
    if (Singleton.instance) return Singleton.instance;
    Singleton.instance = this;
    this.name = name;
  }
}

const a = new Singleton('aaa');
const b = new Singleton('bbb');
console.log(a === b); // true
```

### 3. Proxy 实现
```js
const Singleton = new Proxy(class {
  constructor(name) { this.name = name; }
}, {
  _instance: null,
  construct(target, args) {
    return this._instance ||= new target(...args);
  }
});
```

## 前端常见场景

### 1. 全局状态管理
```js
// Vuex/Redux Store 本质是单例
class Store {
  constructor() {
    if (Store.instance) return Store.instance;
    this.state = {};
    this.listeners = [];
    Store.instance = this;
  }
}
```

### 2. 全局配置
```js
const Config = (() => {
  let config = null;
  return {
    getInstance() {
      if (!config) {
        config = {
          API_BASE: 'https://api.example.com',
          TIMEOUT: 10000,
          // ... 其他配置
        };
      }
      return config;
    }
  };
})();
```

### 3. 弹窗管理器
```js
class ModalManager {
  constructor() {
    if (ModalManager.instance) return ModalManager.instance;
    ModalManager.instance = this;
    this.modals = new Map();
  }

  open(id, content) {
    this.modals.set(id, content);
    // 渲染弹窗
  }

  close(id) {
    this.modals.delete(id);
  }
}
```

### 4. 日志服务
```js
class Logger {
  static getInstance() {
    if (!Logger._instance) {
      Logger._instance = new Logger();
    }
    return Logger._instance;
  }

  log(msg) { console.log(`[LOG] ${msg}`); }
  warn(msg) { console.warn(`[WARN] ${msg}`); }
  error(msg) { console.error(`[ERROR] ${msg}`); }
}

Logger.getInstance().log('应用启动');
```

## 优点
- 全局唯一，避免重复创建
- 节省内存，共享资源

## 缺点
- 隐藏依赖关系，不利于测试
- 全局状态可能导致耦合
