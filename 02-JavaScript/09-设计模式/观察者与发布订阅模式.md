# 观察者模式 Observer / 发布订阅 Pub-Sub

> 一对多依赖，状态变化时自动通知所有订阅者

---

## 观察者模式（直接通知）

```js
class Subject {
  constructor() { this.observers = []; }

  subscribe(observer) {
    this.observers.push(observer);
  }

  unsubscribe(observer) {
    this.observers = this.observers.filter(o => o !== observer);
  }

  notify(data) {
    this.observers.forEach(o => o.update(data));
  }
}

// 使用
const store = new Subject();
store.subscribe({ update: (d) => console.log('A 收到:', d) });
store.subscribe({ update: (d) => console.log('B 收到:', d) });
store.notify('数据更新');
```

## 发布订阅模式（事件中心）

```js
class EventEmitter {
  constructor() { this.events = {}; }

  on(event, callback) {
    (this.events[event] ||= []).push(callback);
    return () => this.off(event, callback); // 返回取消订阅函数
  }

  off(event, callback) {
    this.events[event] = (this.events[event] || []).filter(cb => cb !== callback);
  }

  emit(event, ...args) {
    (this.events[event] || []).forEach(cb => cb(...args));
  }

  once(event, callback) {
    const wrapper = (...args) => {
      callback(...args);
      this.off(event, wrapper);
    };
    return this.on(event, wrapper);
  }
}

// 使用
const bus = new EventEmitter();
bus.on('login', (user) => console.log(`${user} 登录了`));
bus.on('login', () => console.log('触发登录埋点'));
bus.emit('login', 'Tom');
```

## 前端常见场景

### 1. Vue 响应式原理
```js
// 简化版 Vue 2 响应式
function defineReactive(obj, key, val) {
  const dep = []; // 观察者列表
  Object.defineProperty(obj, key, {
    get() {
      if (currentWatcher) dep.push(currentWatcher);
      return val;
    },
    set(newVal) {
      val = newVal;
      dep.forEach(w => w.update()); // 通知所有观察者
    }
  });
}
```

### 2. DOM 事件
```js
// 浏览器原生就是发布订阅
button.addEventListener('click', handler1);
button.addEventListener('click', handler2);
button.click(); // 通知所有订阅者
```

### 3. 跨组件通信
```js
// 事件总线
const eventBus = new EventEmitter();

// 组件 A 发布
eventBus.emit('cart-update', { count: 5 });

// 组件 B 订阅
eventBus.on('cart-update', ({ count }) => {
  badge.textContent = count;
});
```

### 4. Promise 链
```js
// Promise.then 本质上是观察者模式
promise.then(fn1).then(fn2);
// resolve 时按顺序通知
```

## 两者区别

| 特性 | 观察者模式 | 发布订阅模式 |
|---|---|---|
| 耦合度 | 高（Subject 直接持有 Observer） | 低（通过事件中心解耦） |
| 通知方式 | 直接调用 | 事件名匹配 |
| 灵活性 | 低 | 高（支持多事件、通配符） |
| 典型应用 | MVC 模型→视图 | DOM 事件、Node.js EventEmitter |

## 注意事项

- **内存泄漏**：组件销毁时需取消订阅（`off` / `removeEventListener`）
- **顺序依赖**：不要依赖回调的执行顺序
- **性能**：大量订阅者时考虑异步通知或节流
