# 模板方法模式 Template Method

> 在父类中定义算法骨架，子类重写具体步骤

---

## 核心思想

```js
class Beverage {
  // 模板方法：定义流程骨架
  prepare() {
    this.boilWater();
    this.brew();      // 子类实现
    this.pourInCup();
    this.addCondiments(); // 子类实现
  }

  boilWater() { console.log('烧开水'); }
  pourInCup() { console.log('倒入杯中'); }
  brew() { throw new Error('子类必须实现'); }
  addCondiments() { throw new Error('子类必须实现'); }
}

class Tea extends Beverage {
  brew() { console.log('泡茶'); }
  addCondiments() { console.log('加柠檬'); }
}

class Coffee extends Beverage {
  brew() { console.log('冲咖啡'); }
  addCondiments() { console.log('加糖和牛奶'); }
}
```

## 前端常见场景

### 1. 表单提交流程
```js
class FormHandler {
  async submit(form) {
    this.validate(form);       // 子类实现
    const data = this.serialize(form);
    await this.beforeSend(data); // 可选钩子
    const res = await this.request(data);
    this.afterSuccess(res);    // 子类实现
    return res;
  }

  serialize(form) { return new FormData(form); }
  async request(data) { return fetch('/api', { method: 'POST', body: data }); }
}

class LoginForm extends FormHandler {
  validate(form) {
    if (!form.username.value) throw new Error('请输入用户名');
  }
  afterSuccess(res) {
    localStorage.setItem('token', res.token);
    window.location.href = '/dashboard';
  }
}
```

### 2. 组件生命周期
```js
// React 类组件就是模板方法模式
class Component {
  mount() {
    this.componentWillMount?.();
    this.render();
    this.componentDidMount?.();
  }
  update() {
    this.shouldComponentUpdate?.();
    this.componentWillUpdate?.();
    this.render();
    this.componentDidUpdate?.();
  }
}
```

## 优点
- 复用代码：公共逻辑在父类
- 扩展灵活：子类只需关注差异部分
- 流程可控：父类控制执行顺序

## 与策略模式的区别
- **模板方法**：通过继承，算法骨架固定
- **策略模式**：通过组合，运行时可切换策略
