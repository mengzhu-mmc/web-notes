# 原型与原型链

## 面试高频考点

- 原型链的查找机制是什么？
- `prototype`、`__proto__`、`constructor` 三者的关系？
- `instanceof` 的原理是什么？如何手写？
- JS 中有哪些继承方式？各自的优缺点？

---

## 一、三个核心属性

```js
// prototype - 函数特有的属性，指向原型对象
function Person(name) {
  this.name = name;
}

// __proto__ - 每个对象都有，指向其构造函数的 prototype
const person = new Person('张三');
console.log(person.__proto__ === Person.prototype); // true

// constructor - 原型对象上的属性，指回构造函数
console.log(Person.prototype.constructor === Person); // true
```

## 二、原型链的查找机制

当访问一个对象的属性时，JS 引擎会沿着原型链逐级向上查找，直到找到该属性或到达 `null`。

```
实例对象.__proto__  →  构造函数.prototype  →  Object.prototype  →  null
```

```js
function Animal(name) {
  this.name = name;
}
Animal.prototype.eat = function () {
  console.log(this.name + ' is eating');
};

const dog = new Animal('旺财');
// 查找顺序：dog 自身 → dog.__proto__（Animal.prototype）→ 找到 eat
dog.eat(); // 旺财 is eating
```

## 三、instanceof 原理

`instanceof` 用于检测构造函数的 `prototype` 是否存在于实例对象的原型链上。

```js
function myInstanceof(left, right) {
  let prototype = right.prototype;
  let proto = Object.getPrototypeOf(left);

  while (true) {
    if (proto === null) return false;
    if (proto === prototype) return true;
    proto = Object.getPrototypeOf(proto);
  }
}

console.log(myInstanceof([], Array));  // true
console.log(myInstanceof([], Object)); // true
console.log(myInstanceof([], String)); // false
```

## 四、继承方式对比

### 1. 原型链继承

```js
function Parent() {
  this.name = 'parent';
  this.list = [1, 2, 3];
}
function Child() {}
Child.prototype = new Parent();

const c1 = new Child();
const c2 = new Child();
c1.list.push(4);
console.log(c2.list); // [1,2,3,4] ❌ 引用类型共享
```

缺点：引用类型属性共享，无法向父类传参。

### 2. 借用构造函数继承

```js
function Parent(name) {
  this.name = name;
  this.list = [1, 2, 3];
}
function Child(name) {
  Parent.call(this, name); // 核心
}
```

缺点：无法继承父类原型上的方法。

### 3. 组合继承（最常用）

```js
function Parent(name) {
  this.name = name;
  this.list = [1, 2, 3];
}
Parent.prototype.sayName = function () {
  console.log(this.name);
};

function Child(name, age) {
  Parent.call(this, name);   // 第一次调用
  this.age = age;
}
Child.prototype = new Parent();        // 第二次调用
Child.prototype.constructor = Child;
```

缺点：父类构造函数被调用两次。

### 4. 寄生组合继承（最优解）

```js
function Parent(name) {
  this.name = name;
}
Parent.prototype.sayName = function () {
  console.log(this.name);
};

function Child(name, age) {
  Parent.call(this, name);
  this.age = age;
}
// 核心：不调用父类构造函数，只继承原型
Child.prototype = Object.create(Parent.prototype);
Child.prototype.constructor = Child;
```

### 5. ES6 Class 继承（语法糖）

```js
class Parent {
  constructor(name) {
    this.name = name;
  }
  sayName() {
    console.log(this.name);
  }
}

class Child extends Parent {
  constructor(name, age) {
    super(name); // 必须先调用 super()
    this.age = age;
  }
}
// extends 本质上就是寄生组合继承的语法糖
```

### 继承方式总结

| 方式 | 引用类型隔离 | 可传参 | 继承原型方法 | 推荐度 |
| --- | --- | --- | --- | --- |
| 原型链继承 | ❌ | ❌ | ✅ | ⭐ |
| 构造函数继承 | ✅ | ✅ | ❌ | ⭐⭐ |
| 组合继承 | ✅ | ✅ | ✅ | ⭐⭐⭐ |
| 寄生组合继承 | ✅ | ✅ | ✅ | ⭐⭐⭐⭐ |
| ES6 Class | ✅ | ✅ | ✅ | ⭐⭐⭐⭐⭐ |

## 五、原型链继承标准实现（面试标准写法）

> 口诀：**构造函数是爹，原型对象是储物间，实例是儿子**

```js
function Person(name) {
  this.name = name;
}
Person.prototype.sayHi = function() {
  console.log(`我是${this.name}`);
};

function Student(name, score) {
  Person.call(this, name);   // 继承实例属性（借用构造函数）
  this.score = score;
}

Student.prototype = Object.create(Person.prototype); // 继承原型方法，不调用 Person
Student.prototype.constructor = Student;              // 修复 constructor 指向

Student.prototype.showScore = function() {
  console.log(`${this.name}的分数是${this.score}`);
};

const s = new Student('张三', 90);
s.sayHi();      // 我是张三（继承自 Person.prototype）
s.showScore();  // 张三的分数是90
```

**三角关系**：
- `Person.prototype`（储物间）← `Student.prototype`（通过 Object.create 连到储物间）
- `Student`（爹）→ `Student.prototype`（爹的储物间）
- `new Student()`（儿子）→ `__proto__`→ `Student.prototype`

## 六、注意事项

- 原型链越长，属性查找越慢
- 避免修改内置原型（如 `Array.prototype`）
- 实例属性会遮蔽原型上的同名属性
- 原型上的引用类型属性会被所有实例共享
- `Object.create(proto)` 创建新对象并直接设置原型，一步完成，性能最好也最标准
