# 手写深拷贝

## 面试高频考点

- 如何实现一个支持循环引用的深拷贝？
- 为什么用 `Map` 处理循环引用？
- 如何处理 Date、RegExp、Map、Set、Symbol 等特殊类型？
- `structuredClone` 和手写深拷贝的区别？

---

## 一、基础版（处理循环引用）

```js
function deepClone(obj, map = new Map()) {
  // 基本类型直接返回
  if (typeof obj !== 'object' || obj === null) return obj;

  // 循环引用检测
  if (map.has(obj)) return map.get(obj);

  const clone = Array.isArray(obj) ? [] : {};
  map.set(obj, clone); // 先存再递归，防止循环引用死循环

  for (const [key, value] of Object.entries(obj)) {
    clone[key] = deepClone(value, map);
  }

  return clone;
}
```

### 常见踩坑点

| # | 问题 | 类型 |
|---|------|------|
| 1 | 递归时未传 `map`，循环引用检测失效 | 逻辑错误 |
| 2 | `for...of obj` 遍历普通对象（普通对象不可迭代，应用 `Object.entries`） | 运行时错误 |
| 3 | 先递归后存入 map（顺序颠倒，循环引用仍会栈溢出） | 逻辑错误 |

## 二、完整版（支持 Date、RegExp、Map、Set）

```js
function deepClone(obj, map = new Map()) {
  // 基本类型直接返回
  if (typeof obj !== 'object' || obj === null) return obj;

  // 循环引用检测
  if (map.has(obj)) return map.get(obj);

  // 特殊类型处理
  if (obj instanceof Date) return new Date(obj);
  if (obj instanceof RegExp) return new RegExp(obj.source, obj.flags);

  if (obj instanceof Map) {
    const cloneMap = new Map();
    map.set(obj, cloneMap);
    obj.forEach((value, key) => {
      cloneMap.set(deepClone(key, map), deepClone(value, map));
    });
    return cloneMap;
  }

  if (obj instanceof Set) {
    const cloneSet = new Set();
    map.set(obj, cloneSet);
    obj.forEach(value => {
      cloneSet.add(deepClone(value, map));
    });
    return cloneSet;
  }

  // 普通对象和数组
  const clone = Array.isArray(obj) ? [] : Object.create(Object.getPrototypeOf(obj));
  map.set(obj, clone);

  // 处理 Symbol key
  const allKeys = [
    ...Object.keys(obj),
    ...Object.getOwnPropertySymbols(obj)
  ];
  for (const key of allKeys) {
    clone[key] = deepClone(obj[key], map);
  }

  return clone;
}
```

## 三、测试用例

```js
// 循环引用
const a = { name: 'a' };
a.self = a;
const cloned = deepClone(a);
console.log(cloned.self === cloned); // true ✅
console.log(cloned === a); // false ✅

// 特殊类型
const data = {
  date: new Date('2024-01-01'),
  reg: /hello/gi,
  map: new Map([['key', 'value']]),
  set: new Set([1, 2, 3]),
  arr: [1, [2, 3]],
};
const cloned2 = deepClone(data);
cloned2.date.setFullYear(2099);
console.log(data.date.getFullYear()); // 2024 ✅ 原对象不受影响
console.log(cloned2.reg instanceof RegExp); // true ✅
console.log([...cloned2.map.entries()]); // [['key', 'value']] ✅
```

## 四、与 structuredClone 的对比

| 特性 | 手写深拷贝 | `structuredClone`（原生） |
|------|-----------|--------------------------|
| 循环引用 | ✅ 支持（Map 检测） | ✅ 原生支持 |
| Date / RegExp | ✅ 手动处理 | ✅ 原生支持 |
| Map / Set | ✅ 手动处理 | ✅ 原生支持 |
| 函数 | ⚠️ 浅拷贝引用 | ❌ 抛出 DataCloneError |
| Symbol 属性 | ✅ 手动处理 | ❌ 忽略 |
| 兼容性 | 任意环境 | Chrome 98+, Node 17+ |
| 性能 | 中等 | 原生 C++，通常更快 |

```js
// 优先使用 structuredClone（现代环境）
const clone = structuredClone(data);

// 需要兼容旧环境、或处理函数/Symbol，才用手写版
```
