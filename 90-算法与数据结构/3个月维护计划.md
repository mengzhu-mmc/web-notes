**M1-T1 反转链表的第 K 组**
```
用例1：链表 1->2->3->4->5，K=2
预期输出：2->1->4->3->5
（最后1个不足K个，保持原序）

用例2：链表 1->2->3->4->5->6，K=3

（恰好分两组，均翻转）
```
> 边界思考：如果 K=1 呢？链表不变。K > 链表长度呢？不翻转，原样返回。

**M1-T4 打家劫舍：环形 + 每次至少跳两间**
```
用例1：[2, 3, 2]（环形3间房）
预期输出：3
（只能选单独一间，选最大值3）

用例2：[1, 2, 3, 1]（环形4间房）
预期输出：4
（选 index=0 的1 和 index=2 的3，注意环形：0和3相邻，0和1相邻，只有0+2=4合法）
```
> 边界思考：环形时 index 0 和 index n-1 是相邻的，拆成两个子问题分别求最大。

**M2-T2 带过期时间的 LRU 缓存**
```
操作序列（TTL单位：秒，假设时间从0开始）：
  put("a", 1, ttl=3)   // t=0，a在t=3过期
  put("b", 2, ttl=10)  // t=0，b在t=10过期
  get("a")  // t=2 → 返回 1（未过期）
  get("a")  // t=4 → 返回 -1（已过期，a被删除）
  get("b")  // t=4 → 返回 2（b未过期）

预期输出：[1, -1, 2]
```
> 关键：过期 key 既不计入 LRU 命中，也不更新访问顺序，访问到时直接删除。

**M2-T5 字符串解码变体（嵌套 + 变量引用）**
```
用例1：
  变量表：{ x: "ab" }
  输入：2[{x}c]
  预期输出："abcabc"
  （{x}="ab"，展开后为 "abc"，重复2次）

用例2：
  变量表：{ a: "hi", b: "2[{a}]" }（变量值本身不再递归展开，直接字符串）
  输入：3[{a}!]
  预期输出："hi!hi!hi!"
  （{a}="hi"，每组为"hi!"，重复3次）
```
> 关键：遇到 `{x}` 时先查表替换为字符串值，再继续走原来的栈解码流程；变量值本身不递归解析（否则复杂度爆炸）。

</details>

**M3-T4 虚拟列表滚动索引计算**
```
用例1：
  heights = [20, 30, 40, 50, 60]（5个列表项）
  scrollTop = 55
  预期输出：{ index: 2, offset: 5 }
  （前缀和：[0,20,50,90,140,200]，scrollTop=55 落在 [50,90) 即 index=2，偏移 55-50=5）

用例2：
  heights = [100, 100, 100, 100]
  scrollTop = 0
  预期输出：{ index: 0, offset: 0 }
  （滚动为0，第一项，无偏移）
```
> 关键：先构建前缀和数组，再用二分查找第一个 prefixSum[i+1] > scrollTop 的 i。
