## 框架与跨平台

### 🔴 高度关注

**Valdi（Snapchat）**
- 类 React 语法 → 编译为 iOS/Android/macOS 原生应用
- 对比：React Native 走 JS Bridge，Flutter 走 Dart+Skia，Valdi 直接编译为原生 Widget
- 意义：跨平台 UI 框架格局再洗牌，React Native 的有力竞争者
- 链接：https://github.com/Snapchat/Valdi

**VibeGo（AI 辅助 IDE）**
- Vibe Coding 概念落地，Web 端 AI 编码 IDE
- 支持 Claude Code、Gemini CLI 等接入
- 链接：https://github.com/xxnuo/VibeGo

### 🟡 值得跟踪

**WordPress → 纯 JS（my.WordPress.net）**
- WordPress 编译为纯 JS，数据存浏览器本地，无需服务器
- 意义：CMS 向"无服务器化"演进的典型探索

**无头应用（Headless App）概念兴起**
- AI 助手普及后，App 只需提供数据接口，UI 层成为可选项
- 影响：前后端分离进一步深化，BFF（Backend for Frontend）更重要

## CSS 新特性

### 🔴 已进入 Baseline（可放心用）

| 特性 | 说明 | 浏览器支持 |
|------|------|----------|
| **`lh` / `rlh` 单位** | 行高相对单位，1lh = 当前行高，rlh = 根元素行高 | 全部现代浏览器 ✅ |
| **`text-box` 属性** | 解决文字与图标垂直对齐问题 | Chrome/Firefox/Safari ✅ |
| **CSS 嵌套（Native Nesting）** | 类似 Sass 嵌套，无需预处理器 | 全部现代浏览器 ✅ |

### 🟡 正在落地（2026 关注）

| 特性 | 说明 | 来源 |
|------|------|------|
| **CSS Anchor Positioning** | 锚点定位，解决 tooltip/popover 定位难题 | Chrome 125+ |
| **`interestfor` 属性** | 纯 HTML+CSS 实现 hover popover，无需 JS | Chrome 142+ |
| **`dialog` closedBy 属性** | 原生 dialog 支持点击蒙层关闭 | Chrome/Edge ✅，Safari 暂不支持 |
| **`text-decoration-inset`** | 下划线缩进控制 | 逐步落地 |

### 🟢 提案阶段（未来观察）

- **CSS `if()` 函数**：条件样式，减少 CSS 自定义属性冗余
- **`@scope` 规则**：CSS 作用域限定，替代 BEM 的原生方案

## JavaScript 新标准

### 🔴 ES2026 重大进展

**Temporal API（正式进入 ES2026 🎉）**
- 历时 9 年，终于标准化！取代饱受诟病的 `Date` 对象
- 核心优势：不可变类型、时区安全、精确持续时间计算
- 关键类：`Temporal.PlainDate`、`Temporal.ZonedDateTime`、`Temporal.Duration`

```javascript
// Temporal API 示例
const now = Temporal.Now.zonedDateTimeISO('Asia/Shanghai');
console.log(now.toString()); // 2026-03-31T14:00:00+08:00[Asia/Shanghai]

const duration = Temporal.Duration.from({ days: 30 });
const future = now.add(duration);
```

### 🟡 新 API 值得关注

| API | 说明 | 状态 |
|-----|------|------|
| **`Promise.try()`** | 统一同步/异步函数的错误处理 | ES2025 ✅ |
| **`Promise.withResolvers()`** | 暴露 resolve/reject 到外部 | ES2024 ✅ |
| **`Array.fromAsync()`** | 从异步迭代器创建数组 | ES2024 ✅ |
| **正则 `y`（sticky）标志** | 粘性匹配，从 lastIndex 严格匹配 | ES6 标准，少有人用 |
| **`scheduler.yield()`** | 主动让出主线程（优化 INP） | Chrome 115+ |

## 浏览器 & Web 标准

### 🔴 2026 Q1 重要落地

**Chrome 147：Element-scoped View Transitions**
- `element.startViewTransition()` 支持局部、可并发的视图过渡
- 解决了 document 级别不能并发、fixed 定位错位等痛点
- 场景：多列表独立动效、局部组件切换

**Chrome 146：Scoped Custom Element Registries**
- 解决微前端场景下 Web Components 命名冲突
- `new CustomElementRegistry()` 创建独立注册表

**HTML `focusgroup` 属性（Origin Trial）**
- 声明式键盘导航，替代 "roving tabindex" 大量 JS 代码
- 由微软通过 OpenUI 社区组提出，a11y 重要进展

### 🟡 API 演进

| API | 变化 | 影响 |
|-----|------|------|
| **WebTransport** | 全面落地（Chrome/Firefox/Safari） | 实时通信新选择，可替代 WebSocket |
| **Server Push（HTTP/2）** | Chrome 106+ 已移除 | 避免使用，改用 Early Hints |
| **`moveBefore()` DOM API** | 替代 `insertBefore`，不触发重渲染 | Web Components 开发利器 |
| **View Transitions API** | 元素级支持（Chrome 147）| 动效开发进一步简化 |

## 工程化与工具

### 🔴 高优先级

**Chokidar（文件监听）**
- Node.js 文件系统监听，比原生 `fs.watch` 更可靠
- 跨平台一致、内置防抖、支持 rename 事件
- Vite/webpack 热更新底层依赖
- 链接：https://github.com/paulmillr/chokidar

**私有 NPM 新选择**
- **Npflared**：基于 Cloudflare Worker 的轻量私有 NPM 镜像
- 对比：Verdaccio（Node.js 本地）、Nexus（企业级）

### 🟡 安全与供应链

**AI 工具链安全问题升温（重要！）**
- Cline 项目遭 GitHub Issue 提示词注入攻击，恶意版本推送到 npm
- RAG 系统文档投毒（Document Poisoning）成新攻击面
- 建议：审查 AI 编码工具的权限范围，开启 npm 包审计

**AI 复刻开源项目（vinext 事件）**
- Cloudflare 用 AI 一周复刻 Next.js，$1100 Token，构建速度 4x
- 结论：**测试用例**才是真正的技术护城河，代码本身不再是壁垒

### 🟢 编码体验提升

| 工具 | 说明 |
|------|------|
| **RaTeX** | 纯 Rust 的 KaTeX 数学渲染引擎 |
| **AICheck** | 离线检测内容是否由 AI 生成（Rust CLI）|
| **TypeScript 简洁之书** | 开源 TS 教程，中文版免费 |

## AI 与前端融合

### 🔴 已深度影响前端工作流

**AI 编程日常化**
- 代码生成已成日常，开发者关注点转向「如何清晰描述需求」
- 多 Agent 协同开发模式逐渐成为工程化趋势
- 前端测试和文档的价值被重新评估（测试 > 代码）

**大模型本质认知（重要）**
- 大模型本质是统计规律匹配，**训练语料质量决定能力上限**
- 用五种小众语言测试主流模型：平均正确率仅 3.8%（Python 可达 90%）
- 对前端工程师的启示：掌握底层原理，不被 AI 完全替代

### 🟡 值得关注的新应用

| 方向 | 代表项目/技术 | 说明 |
|------|-------------|------|
| **AI 无头应用** | 架构模式 | UI 层可选，AI 直接操作数据接口 |
| **Page Agent** | alibaba/page-agent | 自然语言操作网页的 JS 库 |
| **Canvas 数据压缩** | 原创技术方案 | 利用 canvas 将任意数据编码为图片压缩传输 |
| **微信龙虾接口** | WeClaw/WeChatBot 等 | AI 机器人向微信发消息，多个开源项目接入 |

## 📊 Q1 综合趋势总结

### 上升趋势 ⬆️
1. **原生 Web 标准替代框架**：CSS 嵌套、interestfor、dialog closedBy 等减少 JS 依赖
2. **性能指标升级**：INP 取代 FID，交互响应性成为核心关注点
3. **实时通信演进**：WebTransport 和 HTTP/3 QUIC 全面落地
4. **AI 工具链安全**：供应链攻击成新兴威胁

### 降温趋势 ⬇️
1. **HTTP/2 Server Push**：已被主流浏览器弃用
2. **FID 指标**：2024 年退役，迁移到 INP
3. **复杂 CSS 预处理器**：原生 CSS 能力追上 Sass/Less

### 持续观察 👀
1. **Temporal API 落地进度**：已进 ES2026，等待浏览器全面支持
2. **CSS Anchor Positioning 普及**：解决 tooltip 定位痛点
3. **AI 无头应用架构**：颠覆传统前端开发模式？
