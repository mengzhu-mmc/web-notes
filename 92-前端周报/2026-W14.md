4. **CSS 锚点定位（`anchor-name`/`position-anchor`）**：不用 JS 计算坐标就能将元素"锚定"在另一个元素旁边

### JavaScript 新 API

- **`Promise.try(fn)`**：统一包装同步和异步函数的执行，不再需要 `try{}catch{}` 包一层 Promise
- **`Promise.withResolvers()`**：返回 `{ promise, resolve, reject }`，无需在 Promise 构造函数外暴露 resolve/reject

### WebTransport：下一代实时通信

- 基于 **HTTP/3 + QUIC** 协议
- 支持**可靠流**（类 TCP）和**不可靠数据报**（类 UDP）两种传输模式
- 相比 WebSocket：无队头阻塞、连接建立更快、多流并发
- 全部现代浏览器已支持，适合：游戏实时同步、直播推流、IoT 低延迟场景

### 1. WebTransport vs WebSocket

**面试常问**：WebSocket 有什么缺点？WebTransport 解决了什么问题？

| 对比维度 | WebSocket | WebTransport |
|---------|-----------|-------------|
| 底层协议 | TCP（via HTTP/1.1） | QUIC（via HTTP/3） |
| 队头阻塞 | 有（TCP 层） | 无（QUIC 多流独立） |
| 数据可靠性 | 仅可靠 | 可靠流 + 不可靠数据报 |
| 连接建立 | 需 HTTP Upgrade 握手 | 0-RTT 快速连接 |
| 多路复用 | 单连接单流 | 单连接多流 |
| 浏览器支持 | 全支持 | 现代浏览器全支持 |

**使用场景**：游戏、直播、实时协作（WebTransport）；聊天、通知（WebSocket 仍适用）

### 3. Promise.try() 使用场景

**解决痛点**：某函数可能同步也可能异步返回，统一用 Promise 包装时写法繁琐

```javascript
// 之前的写法
function wrapper(fn) {
  return new Promise((resolve) => resolve(fn())); // 同步错误不会被 catch
  // 或者：
  return Promise.resolve().then(() => fn()); // 正确但绕
}

// Promise.try()（2026 新增）
function wrapper(fn) {
  return Promise.try(fn); // 同步错误也会转为 rejected promise
}
```

**面试考点**：同步异常在 Promise 中的传播、`Promise.resolve()` vs `Promise.try()`

### 4. CSS `interestfor` 属性（Hover Popover）

```html
<!-- 无需 JavaScript！悬停 button 自动显示 tooltip -->
<button interestfor="my-tooltip">悬停我</button>
<div id="my-tooltip" popover>这是提示内容</div>
```

**技术原理**：
- `interestfor` 让元素声明对另一个元素"感兴趣"
- 鼠标悬停触发器 → 浏览器自动显示目标 popover
- 配合 CSS `:popover-open` 伪类可实现复杂样式

**面试方向**：Popover API 全貌（`popover`、`popovertarget`、`interestfor`）
