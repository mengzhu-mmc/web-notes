## 🔧 张鑫旭博客 本周新文

### 🌟 重点推荐：WebTransport API — 下一代 Web 双向通信技术

> 发布于 2026-03-17 | [原文链接](https://www.zhangxinxu.com/wordpress/2026/03/webtransport-api/)

**一句话总结**：基于 HTTP/3 + QUIC 的新一代实时通信 API，解决 WebSocket 的队头阻塞、单一流、网络切换断连等痛点。

#### WebTransport vs WebSocket 核心对比

| 对比维度 | WebSocket | WebTransport |
|---------|-----------|-------------|
| 协议基础 | HTTP/1.1 + TCP | HTTP/3 + QUIC（UDP） |
| 连接建立 | TCP 三次握手，延迟较高 | QUIC 0-RTT/1-RTT，最快100ms |
| 传输模式 | 单一可靠流 | 可靠流 + 不可靠数据报 |
| 队头阻塞 | 有 | 无 |
| 网络切换 | 断开重连 | 无缝迁移（连接ID识别）|
| 适用场景 | 普通聊天、消息推送 | 实时游戏、直播推流、高频数据 |

#### 三大核心特性

1. **双重传输模式**：可靠流（保证有序）+ 不可靠数据报（极低延迟）
2. **多路复用**：一个连接多个并行流，互不阻塞
3. **连接迁移**：Wi-Fi 切 4G 不断连

#### 浏览器支持
- Chrome 97+、Firefox 114+、Safari 26.4+（已全部支持）
- 须在 HTTPS 环境使用
