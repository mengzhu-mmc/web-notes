### [Temporal API]（ECMAScript 2026 / Stage 4）
> JavaScript 终于有了靠谱的日期时间 API——不可变、时区原生支持、纳秒精度，Date 的历史遗留问题一扫而空。

**浏览器支持**：Chrome 134+ ✅ / Firefox 141+ ✅ / Safari 18.2+ ✅

**代码示例**
```javascript
// 创建带时区的精确时间（不再需要手算 offset）
const meeting = Temporal.ZonedDateTime.from({
  timeZone: 'Asia/Shanghai',
  year: 2026, month: 4, day: 14,
  hour: 15, minute: 30,
});

const now = Temporal.Now.plainDateTimeISO();
const later = now.add({ hours: 2, minutes: 30 });

// 差值计算（链式 API）
const duration = later.since(now);
console.log(duration.round({ largestUnit: 'hours' })); // PT2H30M

// 不再有月份 0-11 的坑
const date = Temporal.PlainDate.from({ year: 2026, month: 4, day: 14 });
console.log(date.month); // 4 ✅ 不再是 3
```

### [`using` / `await using`]（ECMAScript 2026）
> 新增 `using` 和 `await using` 关键字，同步/异步资源自动释放，无需手动写 try...finally 清理逻辑。

**浏览器支持**：Chrome 134+ ✅ / Firefox 141+ ✅ / Safari ❌（Node.js 20+ 已支持）

**代码示例**
```javascript
// 同步资源：数据库连接、文件句柄
class DBConnection {
  constructor(url) { this.url = url; }
  [Symbol.dispose]() {
    console.log(`关闭连接: ${this.url}`);
  }
}

function query() {
  using db = new DBConnection('mysql://localhost/app');
  // 用完自动调用 db[Symbol.dispose()]，无需 finally
}

// 异步资源：网络流
async function fetchData() {
  await using stream = openNetworkStream('https://api.example.com/data');
  // 异步清理网络连接
}
```

### [Error.isError()]（ECMAScript 2026）
> 跨 realm（iframe/worker）安全地判断一个值是否为 Error 对象，`instanceof Error` 在沙盒场景下会误判，Error.isError() 才是正确的做法。

**浏览器支持**：Chrome 134+ ✅ / Firefox 138+ ✅ / Safari ❌

**代码示例**
```javascript
// 传统方法：跨 realm 不可靠
const frame = document.createElement('iframe');
document.body.appendChild(frame);
// iframe 里的 Error 实例用 instanceof 判断会失败
frame.contentWindow.postMessage(error, '*');

// 新方法：跨 realm 安全
Error.isError(new TypeError('oops'));     // true ✅
Error.isError({ name: 'Error' });          // false ✅
Error.isError('not an error');            // false ✅
```

### [Uint8Array to/fromBase64 / to/fromHex]（ECMAScript 2026）
> Uint8Array 新增四个静态方法，直接完成与 Base64/十六进制字符串的双向转换，无需手写 btoa/atob 或第三方库。

**浏览器支持**：Chrome 140+ ✅ / Firefox 133+ ✅ / Safari 18.2+ ✅ / Opera ❌

**代码示例**
```javascript
// Base64 互转
const bytes = Uint8Array.fromBase64('V1NjaG9vbHMgMTIz');
console.log(Array.from(bytes)); // [91, 116, 156, 134, 138, 37, 179, 93, 183]

const encoded = new Uint8Array([91, 116, 156]).toBase64();
console.log(encoded); // 'W3NjaA=='

// 十六进制互转
const hex = Uint8Array.fromHex('5b749c868a25b3');
console.log(hex); // Uint8Array([91, 116, 156, 134, 138, 37, 179])

const back = hex.toHex();
console.log(back); // '5b749c868a25b3'
```

### [Array.fromAsync()]（ECMAScript 2026 / Stage 4）
> 从异步可迭代对象、Promise 数组、类数组创建数组，一行替代原来的手动 await + Array.from 套路。

**浏览器支持**：Chrome 121+ ✅ / Firefox 115+ ✅ / Safari 16.4+ ✅

**代码示例**
```javascript
// 异步生成器
async function* fetchUsers(ids) {
  for (const id of ids) {
    const res = await fetch(`/api/user/${id}`);
    yield await res.json();
  }
}

async function main() {
  const ids = [1, 2, 3, 4, 5];
  
  // 旧写法：手动 await + from
  const users = await Promise.all(ids.map(id => fetchUser(id)));
  
  // 新写法：一行搞定
  const users = await Array.fromAsync(fetchUsers(ids));
  console.log(users); // [{id:1,...}, {id:2,...}, ...]
}
```

### [CSS :has() 伪类]（全浏览器支持）
> CSS 历史上首次"向上看"的选择器——父元素可以基于子元素状态来匹配，终结大量基于子元素状态变更样式的 JavaScript 代码。

**浏览器支持**：Chrome 105+ ✅ / Firefox 121+ ✅ / Safari 15.4+ ✅

**代码示例**
```css
/* 当表单字段无效时，给整个 field 标红 */
.form-field:has(input:invalid) {
  border-color: red;
}

/* 购物车有商品时隐藏空状态 */
.cart:not(:has(.cart-item)) .empty-state {
  display: block;
}

/* 卡片含视频时特殊样式 */
.card:has(video) {
  background: linear-gradient(135deg, #1a1a2e, #16213e);
  color: white;
}

/* 表单全部填写后解锁提交按钮 */
.form:not(:has(input:required:invalid)) .submit-btn {
  opacity: 1;
  cursor: pointer;
}
```
