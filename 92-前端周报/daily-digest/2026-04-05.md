### 1️⃣ Promise.try()（ECMAScript 2027）

> 统一处理同步和异步函数的 Promise 包装，解决同步错误无法被 catch 的问题

**浏览器支持**：Chrome 134+ / Firefox 136+ / Safari 17.5+（部分支持，需 polyfill）

**核心原理**
`Promise.try()` 是 ECMAScript 2027 提案中的新方法，用于将函数调用"提升"到 Promise 上下文中。它解决了 `Promise.resolve(func())` 无法捕获同步错误的痛点，让错误处理更统一。

**代码示例**
```javascript
// 之前的痛点：同步错误无法被 catch
function wrapper(fn) {
  return Promise.resolve(fn()); // 如果 fn 同步抛错，不会被 catch
}

// Promise.try() 解决方案
function wrapper(fn) {
  return Promise.try(fn); // 同步错误也会转为 rejected promise
}

// 实际使用
Promise.try(() => JSON.parse('{"a":1}'))
  .then(data => console.log(data))
  .catch(err => console.error('解析失败:', err));

// 结合 catch/finally 实现类似同步的错误处理
Promise.try(action)
  .then(result => console.log(result))
  .catch(error => console.error(error))
  .finally(() => console.log('Done'));
```

- [ECMAScript 2027 Proposal](https://tc39.es/ecma262/multipage/control-abstraction-objects.html#sec-promise.try)

### 2️⃣ WebTransport API（HTTP/3 + QUIC）

> 基于 HTTP/3 的下一代实时通信 API，解决 WebSocket 队头阻塞问题

**浏览器支持**：Chrome 97+ / Firefox 114+ / Safari 16.4+（全平台支持）

**核心原理**
WebTransport 基于 Google 的 QUIC 协议（HTTP/3），提供低延迟的双向通信。相比 WebSocket：
- **无队头阻塞**：QUIC 多流独立，单流失败不影响其他流
- **更快连接**：0-RTT 快速连接，无需 HTTP Upgrade 握手
- **双模式传输**：可靠流（类 TCP）+ 不可靠数据报（类 UDP）

**代码示例**
```javascript
// 建立 WebTransport 连接
const url = 'https://example.com:4999/wt';
const transport = new WebTransport(url);

// 等待连接就绪
await transport.ready;
console.log('WebTransport 已连接');

// 发送不可靠数据报（适合游戏状态同步）
const writer = transport.datagrams.writable.getWriter();
writer.write(new Uint8Array([65, 66, 67]));

// 接收数据报
const reader = transport.datagrams.readable.getReader();
while (true) {
  const { value, done } = await reader.read();
  if (done) break;
  console.log('收到数据:', value); // Uint8Array
}

// 创建可靠双向流（适合聊天、文件传输）
const stream = await transport.createBidirectionalStream();
const writable = stream.writable.getWriter();
writable.write(new TextEncoder().encode('Hello!'));

// 监听连接关闭
await transport.closed;
```

**使用场景**
- 🎮 游戏实时同步（数据报模式）
- 📹 直播推流（流模式）
- 🤝 协作编辑（流模式）
- 📱 IoT 低延迟通信

**参考链接**
- [MDN: WebTransport API](https://developer.mozilla.org/en-US/docs/Web/API/WebTransport_API)
- [Chrome Developers: WebTransport](https://developer.chrome.com/docs/capabilities/web-apis/webtransport)

### 3️⃣ HTML dialog closedBy 属性（2026 新增）

> 原生弹窗元素支持声明式关闭行为，无需 JavaScript

**浏览器支持**：Chrome 133+ / Firefox 136+ / Safari 17.5+

**核心原理**
`closedBy` 属性让开发者声明弹窗的关闭方式，三种取值：
- `any`：任意方式关闭（点击蒙层 + Esc + 代码关闭）
- `closerequest`：平台特定方式（Esc）+ 代码关闭
- `none`：仅代码关闭

**代码示例**
```html
<!-- 点击蒙层也能关闭 -->
<dialog id="dialog1" closedby="any">
  <p>点击外部或按 Esc 都能关闭</p>
  <button onclick="dialog1.close()">关闭</button>
</dialog>

<!-- 只能按 Esc 或点击按钮关闭 -->
<dialog id="dialog2" closedby="closerequest">
  <p>只能按 Esc 或点击按钮关闭</p>
  <button onclick="dialog2.close()">关闭</button>
</dialog>

<!-- 只能代码关闭（适合重要确认） -->
<dialog id="dialog3" closedby="none">
  <p>必须确认后才能关闭</p>
  <button onclick="dialog3.close()">确认</button>
</dialog>

<script>
  // 打开模态弹窗
  document.getElementById('dialog1').showModal();
</script>
```

**参考链接**
- [MDN: dialog 元素](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/dialog)

### 4️⃣ Popover API + interestfor 属性

> 纯 HTML+CSS 实现悬停触发弹层，无需 JavaScript 事件监听

**浏览器支持**：Chrome 114+ / Firefox 125+ / Safari 17+

**核心原理**
`interestfor` 属性声明元素对另一个元素的"兴趣"，鼠标悬停时浏览器自动显示目标 popover。配合 CSS `:popover-open` 伪类可实现复杂样式。

**代码示例**
```html
<!-- 无需 JavaScript！悬停按钮自动显示 tooltip -->
<button interestfor="tooltip1">悬停我</button>
<div id="tooltip1" popover="hint">
  <p>这是提示内容</p>
</div>

<!-- 带延迟的悬停提示 -->
<style>
  #tooltip1 {
    interest-delay: 300ms; /* 悬停 300ms 后显示 */
  }
  #tooltip1:popover-open {
    /* 弹出时的样式 */
    opacity: 1;
    transform: translateY(0);
  }
</style>

<!-- 控制按钮 -->
<button popovertarget="menu1">打开菜单</button>
<div id="menu1" popover="auto">
  <button popovertarget="menu1" popovertargetaction="hide">关闭</button>
</div>
```

**相关 CSS 属性**
- `interest-delay`：悬停延迟
- `:interest-source`：悬停源选择器
- `:interest-target`：目标元素选择器
- `:popover-open`：弹出状态伪类

**参考链接**
- [MDN: Popover API](https://developer.mozilla.org/en-US/docs/Web/API/Popover_API)

### 5️⃣ CSS 锚点定位（Anchor Positioning）

> 纯 CSS 实现元素相对定位，无需 JavaScript 计算坐标

**浏览器支持**：Chrome 125+ / Firefox 131+ / Safari 17.5+

**核心原理**
CSS 锚点定位让元素"锚定"在另一个元素旁边，自动处理溢出和翻转。常用于 tooltip、dropdown、popover 等浮动组件。

**代码示例**
```css
/* 定义锚点 */
.anchor-button {
  anchor-name: --my-anchor;
}

/* 锚定到按钮下方 */
.tooltip {
  position: absolute;
  position-anchor: --my-anchor;
  top: anchor(bottom); /* 锚点底部 */
  left: anchor(center); /* 锚点中心 */
  transform: translateX(-50%);
}

/* 带溢出处理 */
.dropdown {
  position: absolute;
  position-anchor: --my-anchor;
  top: anchor(bottom);
  left: anchor(left);
  /* 自动翻转：如果下方空间不足，翻到上方 */
  position-try-options: flip-block;
}
```

**相关属性**
- `anchor-name`：定义锚点名称
- `position-anchor`：引用锚点
- `anchor()`：锚点位置函数
- `position-try-options`：溢出处理策略

### 6️⃣ Invoker Commands API（command + commandfor）

> 纯 HTML 控制弹窗，替代 JavaScript 事件监听

**浏览器支持**：Chrome 133+ / Firefox 136+ / Safari 17.5+

**代码示例**
```html

  <p>这是弹窗内容</p>
```
