# 手写深拷贝

## 面试高频考点

- 如何实现一个支持循环引用的深拷贝？
- 为什么用 `Map` 处理循环引用？
- 遍历普通对象应该用 `for...in` 还是 `for...of`？

---

## 实现

```js
const deepClone = (obj, map = new Map()) => {
  // 基本类型直接返回
  if (typeof obj !== 'object' || obj === null) {
    return obj;
  }

  // 循环引用检测
  if (map.has(obj)) {
    return map.get(obj);
  }

  const objClone = Array.isArray(obj) ? [] : {};
  map.set(obj, objClone);

  for (let [key, value] of Object.entries(obj)) {
    objClone[key] = deepClone(value, map); // 递归时必须传 map
  }

  return objClone;
};
```

## 常见错误

| # | 问题 | 类型 |
| --- | --- | --- |
| 1 | 变量拼写错误（如 `ojb` 写成 `obj`） | 拼写错误 |
| 2 | `for...of obj` 遍历普通对象（普通对象不可迭代，应用 `Object.entries`） | 逻辑错误 |
| 3 | 递归时未传 `map`，导致循环引用检测失效 | 逻辑错误 |

## 测试用例

```js
// 基本类型
console.log(deepClone(42)); // 42
console.log(deepClone(null)); // null

// 嵌套对象
const nested = { a: { b: { c: 42 } } };
const clonedNested = deepClone(nested);
clonedNested.a.b.c = 999;
console.log(nested.a.b.c); // 42 ✅ 原对象不受影响

// 循环引用
const circular = { name: 'circular' };
circular.self = circular;
const clonedCircular = deepClone(circular);
console.log(clonedCircular.self === clonedCircular); // true ✅
console.log(clonedCircular === circular); // false ✅
```

## 覆盖场景总结

| 测试项 | 目的 |
| --- | --- |
| 基本类型 | 验证非对象直接返回 |
| 普通对象 / 数组 | 验证基础克隆功能 |
| 深层嵌套 | 验证递归是否正确 |
| 循环引用 | 验证 Map 是否生效 |
| 深拷贝独立性 | 验证修改克隆不影响原对象 |
