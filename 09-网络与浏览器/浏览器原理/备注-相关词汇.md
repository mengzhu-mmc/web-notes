## 渲染管线关键概念

### Reflow（回流 / 重排）
**触发时机**：改变了影响布局的 CSS 属性（width、height、margin、padding、position、font-size 等）或读取布局属性（offsetWidth、clientHeight、getBoundingClientRect 等）。

**代价**：主线程重新计算所有受影响元素的几何信息，耗时高，会影响子元素和相邻元素。

```js
// 触发 Reflow 的操作（避免在循环中使用）
el.offsetWidth   // 读取布局属性
el.offsetHeight
el.clientTop
el.getBoundingClientRect()
el.scrollTop
window.getComputedStyle(el)
```

### Repaint（重绘）
**触发时机**：改变了不影响布局的视觉属性（color、background、box-shadow、outline 等）。

**代价**：比 Reflow 轻，但仍需 Paint 步骤，不触发 Layout。

> ⚡ **口诀**：Reflow 一定触发 Repaint；Repaint 不一定触发 Reflow。

### Compositing（合成）
**触发时机**：使用 `transform`、`opacity`、`filter`（部分）、`will-change` 提升为合成层的元素发生变化。

**代价最低**：整个过程由 GPU 完成，完全不占用主线程，不触发 Layout 和 Paint。

> ✅ **最优化**：动画尽量只使用 `transform` 和 `opacity`，让浏览器走 Compositing 路径。

```css
/* ✅ 只触发 Compositing（GPU 处理）*/
.animate { transform: translateX(100px); opacity: 0.5; }

/* ❌ 触发 Layout + Paint + Composite（昂贵）*/
.animate { left: 100px; }
```

## 图层系统

### Paint Layer（绘制层）
浏览器将页面内容分成多个 Paint Layer 分别绘制，减少无效绘制区域。每个 `position: relative/absolute/fixed` 或 `overflow` 的元素可能创建独立 Paint Layer。

### Composite Layer（合成层 / 提升层）
满足特定条件的元素会被浏览器提升为独立的 **GPU 合成层**，该层的变化（transform/opacity）由 GPU 直接处理，不影响其他层。

**创建合成层的条件**：
- `transform: translateZ(0)` 或 `will-change: transform`（常用黑魔法）
- `position: fixed`
- `<video>` / `<canvas>` / `<iframe>`
- CSS 动画中使用 `transform`/`opacity`
- `backface-visibility: hidden`

```css
/* 强制提升为合成层（慎用，占用 GPU 内存）*/
.will-animate {
  will-change: transform;
}
```

### Stacking Context（层叠上下文）
决定元素在 Z 轴上的渲染顺序。创建新层叠上下文的条件：
- `position: absolute/relative/fixed/sticky` + `z-index` 非 `auto`
- `opacity` < 1
- `transform` 非 `none`
- `filter` 非 `none`
- `will-change` 指定以上任意属性
- `isolation: isolate`

## 进程与线程

| 进程/线程 | 职责 |
|-----------|------|
| Browser Process | 浏览器主进程，管理 UI、网络、存储 |
| Renderer Process | 每个 Tab 独立的渲染进程（Chrome 多进程架构）|
| GPU Process | 处理 GPU 指令，合成图层 |
| Main Thread | 渲染进程主线程：解析 HTML/CSS、执行 JS、Layout、Paint |
| Compositor Thread | 合成线程：处理滚动、合成图层（与主线程独立）|
| Raster Thread | 光栅化线程：将 Paint 指令转为像素 |

## 其他常用词汇

| 英文 | 中文 | 说明 |
|------|------|------|
| Critical Rendering Path | 关键渲染路径 | HTML→DOM→CSSOM→RenderTree→Layout→Paint |
| Render Blocking | 渲染阻塞 | CSS/JS 阻止页面首次渲染 |
| Layout Thrashing | 布局抖动 | 循环中交替读写 DOM，导致多次强制同步布局 |
| FOUC | 无样式内容闪烁 | Flash of Unstyled Content，CSS 加载前内容短暂裸露 |
| TTI | Time to Interactive | 页面可交互时间 |
| TTFB | Time to First Byte | 从请求发出到收到第一个字节的时间 |
| Tree Shaking | 树摇 | 打包时删除未使用的代码 |
| Code Splitting | 代码分割 | 按需加载，减少首屏包大小 |
