# HTTPS

### 一、HTTP连接建立的流程

![在这里插入图片描述](https://img-blog.csdnimg.cn/20181121201424403.png?x-oss-process=image/watermark,type_ZmFuZ3poZW5naGVpdGk,shadow_10,text_aHR0cHM6Ly9ibG9nLmNzZG4ubmV0L2tzd2tseQ==,size_16,color_FFFFFF,t_70)

开始加密通信之前，客户端和服务器首先必须建立连接和交换参数，这个过程叫做握手

1. 首先客户端向服务端发送一个请求，里面主要包括：一个客户端生成的随机数，以及客户端支持的加密算法（都是对称加密算法）。

2. 服务端收到请求后从中选出一组加密算法和HASH算法，并将自己的数字证书、以及一个自己生产的随机数发送给客户端
3. 客户端收到回复后，验证证书是否有效，如果有效就生成一个随机数，然后用证书中所包含的公钥加密这个随机数生产密文，并且用客户端刚才挑选的HASH算法对这个 随机数进行加密，生成的hash值称之为校验值1，然后将密文和校验值1一起发送给服务端。
4. 服务端收到后，用自己的私钥解密处随机数，然后用HASH算法对这个随机数进行加密得到校验值2，如果校验值1和校验值2相同则说明消息未被篡改，可以信任。最后，使用该随机数，加上之前第二步中选择的加密算法，加密一段握手信息，发给客户端，同时HASH值也带上。
5. 客户端收到服务端的消息后，首先检测消息是否为握手消息，然后计算HASH值是否与发回的消息一致。
6. 握手结束后，客户端和服务端使用握手阶段产生的随机数以及挑选出来的算法进行对称加密的传输。
