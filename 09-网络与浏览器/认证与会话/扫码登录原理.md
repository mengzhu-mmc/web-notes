# 扫码登录原理

> 现代移动端扫码登录（二维码登录）的完整流程与实现原理

---

## 面试高频考点

- 扫码登录的完整流程是什么？（PC → 服务端 → App → 服务端 → PC）
- PC 端如何感知扫码状态变化？（轮询 vs WebSocket）
- 二维码里存的是什么？（唯一 token，不是账号密码）
- 与传统密码登录的核心区别？
- token 超时处理？二维码刷新？

---

## 一、完整流程（5步）

```
PC 端                    服务端                    App
  |                         |                         |
  |--- 请求生成二维码 ------>|                         |
  |<-- 返回二维码+token -----|                         |
  |                         |                         |
  | 显示二维码，轮询/WS监听  |                         |
  |                         |                         |
  |                         |<-- 扫码+携带token+账号 --|
  |                         |--- 标记token状态:已确认  |
  |                         |                         |
  |--- 轮询/WS:token状态? -->|                         |
  |<-- 状态:已确认 ----------|                         |
  |                         |                         |
  |--- 用token换登录凭证 --->|                         |
  |<-- 返回 sessionId/JWT ---|                         |
  | 登录成功                 |                         |
```

### 详细说明

1. **PC 请求二维码**：服务端生成唯一 token（UUID 等），与该 token 绑定"待扫码"状态，返回二维码图片（内含 token）
2. **App 扫描二维码**：App 解析二维码获取 token，连同已登录的 App 账号信息，向服务端发送"确认登录"请求
3. **服务端更新状态**：服务端将 token 对应状态更新为"已确认"，记录对应的账号
4. **PC 感知状态变化**：PC 端通过轮询或 WebSocket 监听 token 状态，发现"已确认"后，用 token 换取正式登录凭证（Session/JWT）
5. **颁发凭证，登录成功**：服务端验证 token 有效性，返回登录凭证，PC 端存储并完成登录

## 二、与传统密码登录对比

| 维度 | 密码登录 | 扫码登录 |
|------|---------|---------|
| 凭证传输 | 密码在网络传输（有泄露风险） | 密码从不经过网络 |
| 登录发起方 | PC 端输入密码 | App 端确认授权 |
| 安全性 | 受钓鱼攻击威胁 | 二维码是一次性的，防钓鱼 |
| 使用前提 | 记得密码 | 手机已登录 App |
| 适用场景 | 通用 | 已有手机 App 认证体系的产品 |

## 三、代码示例

### PC 端轮询监听状态

```js
// 方式一：定时轮询
async function pollQRCodeStatus(token) {
    const timer = setInterval(async () => {
        const res = await fetch(`/api/qrcode/status?token=${token}`);
        const { status, sessionId } = await res.json();

        if (status === 'confirmed') {
            clearInterval(timer);
            // 保存登录凭证
            localStorage.setItem('sessionId', sessionId);
            router.push('/home');
        } else if (status === 'expired') {
            clearInterval(timer);
            refreshQRCode(); // 二维码过期，刷新
        }
    }, 2000); // 每2秒轮询一次

    // 超时自动停止（二维码有效期，如 5 分钟）
    setTimeout(() => clearInterval(timer), 5 * 60 * 1000);
}
```

### PC 端 WebSocket 监听状态（更推荐）

```js
// 方式二：WebSocket 实时监听（更实时，无冗余请求）
function listenQRCodeStatus(token) {
    const ws = new WebSocket(`wss://example.com/ws/qrcode?token=${token}`);

    ws.onmessage = (event) => {
        const { status, sessionId } = JSON.parse(event.data);

        if (status === 'scanned') {
            showMessage('已扫描，等待确认...');
        } else if (status === 'confirmed') {
            ws.close();
            localStorage.setItem('sessionId', sessionId);
            router.push('/home');
        } else if (status === 'expired') {
            ws.close();
            refreshQRCode();
        }
    };

    ws.onerror = () => {
        // 降级为轮询
        pollQRCodeStatus(token);
    };
}
```

### 服务端示意（Node.js / Express）

```js
// 生成二维码
app.get('/api/qrcode/generate', (req, res) => {
    const token = uuid.v4();
    qrCodeStore.set(token, { status: 'pending', expiresAt: Date.now() + 5 * 60 * 1000 });
    res.json({ token, qrUrl: `https://example.com/scan?token=${token}` });
});

// App 扫码确认
app.post('/api/qrcode/confirm', authRequired, (req, res) => {
    const { token } = req.body;
    const record = qrCodeStore.get(token);
    if (!record || Date.now() > record.expiresAt) {
        return res.status(400).json({ error: 'token expired' });
    }
    record.status = 'confirmed';
    record.userId = req.user.id; // 记录已登录的账号
    res.json({ ok: true });
});

// PC 端查询状态
app.get('/api/qrcode/status', (req, res) => {
    const { token } = req.query;
    const record = qrCodeStore.get(token);
    if (!record) return res.json({ status: 'expired' });
    if (Date.now() > record.expiresAt) {
        qrCodeStore.delete(token);
        return res.json({ status: 'expired' });
    }
    if (record.status === 'confirmed') {
        const sessionId = createSession(record.userId);
        qrCodeStore.delete(token); // 用完即销毁
        return res.json({ status: 'confirmed', sessionId });
    }
    res.json({ status: record.status });
});
```

## 面试要点总结

- 二维码里存的是**一次性 token**，不是账号密码
- 核心链路：PC 生成 → App 确认 → 服务端标记 → PC 感知 → 颁发凭证
- PC 感知方式：轮询（简单）或 WebSocket（更实时）
- 安全性高：密码从不经过网络，token 一次性且有过期时间
- 二维码过期需刷新（通常 3~5 分钟有效期）
