## 第一类：概率与期望

### 1. 赛马问题

**来源：** 谷歌 / 微软 / 字节跳动（经典高频题）

**题目：**  
25 匹马，5 条赛道，每次最多同时跑 5 匹，不能计时，只能知道同组内的名次。至少比赛几次才能确定最快的前 3 名？

**答案：** 7 次

**解题思路：**

1. **第 1~5 次**：将 25 匹分成 5 组（A/B/C/D/E），每组各跑一次，得到组内名次。共 5 次。
2. **第 6 次**：取每组第 1 名（A1、B1、C1、D1、E1）同场竞技。假设结果为 A1 > B1 > C1 > D1 > E1。
   - D 组和 E 组所有马全部淘汰（因为它们连各自组第 1 名都比不过排名第 4、5 的马）
   - C 组只有 C1 可能进前 3（C2、C3 不可能）
   - B 组只有 B1、B2 可能进前 3（B3 不可能）
   - A 组 A1 是第 1（确定），A2、A3 也可能进前 3
   - **A1 已确认第 1，不需再跑**
3. **第 7 次**：剩余候选马最多 5 匹：A2、A3、B1、B2、C1，跑最后一次，取前 2 名，与 A1 合并即为前 3 名。

**考查意图：**  
- 逻辑推理与分治思想
- 能否系统地缩小候选集而不浪费比赛次数
- 是否能主动分析每一步的信息增益

### 2. 烧绳子计时

**来源：** 微软 / 谷歌（经典）

**题目：**  
两根不均匀的绳子，每根从头烧到尾各需 1 小时（燃烧速度不均匀，不能折叠量长度）。如何用这两根绳子准确计时 45 分钟？

**答案：** 同时点燃绳 A 的两端 + 绳 B 的一端；当绳 A 烧完时（30 分钟），立刻点燃绳 B 的另一端；绳 B 烧完时即经过了 45 分钟。

**解题思路：**

| 时刻 | 绳 A 状态 | 绳 B 状态 | 已过时间 |
|------|-----------|-----------|----------|
| 0 min | 两端点燃 | 一端点燃 | 0 |
| 30 min | 烧完 | 还剩 30 min | 30 min |
| 点燃绳 B 另一端 | — | 两端燃烧，剩 15 min | 30 min |
| 45 min | — | 烧完 | **45 min ✅** |

**关键洞察：** 两端同时燃烧，烧完所需时间是原来的一半（无论绳子粗细不均匀如何）。

**考查意图：**  
- 逆向思维：把「燃烧时间」转化为「信息载体」
- 能否利用「两端点燃 = 计时减半」这个隐藏工具

### 3. 生男生女问题

**来源：** 谷歌 / 美团（反直觉概率题）

**题目：**  
某国规定：每个家庭生到第一个男孩就停止生育，否则继续生。最终全国男女比例是多少？

**答案：** 1 : 1（男女各占一半）

**解题思路：**

每次生育都是独立的随机事件，P(男) = P(女) = 1/2，与家庭策略无关。

验证：
- 1/2 的家庭：直接生男孩，只有 1 个孩子（男）
- 1/4 的家庭：先女后男，2 个孩子（1女1男）
- 1/8 的家庭：女女男，3 个孩子（2女1男）
- ……

期望男孩总数 = 1（每家只有 1 个男孩）  
期望女孩总数 = 0×(1/2) + 1×(1/4) + 2×(1/8) + … = **1**

所以男女比例 = 1 : 1。

**误区：** 直觉认为「生男才停，所以男孩少」——这是混淆了「策略」与「单次概率」。

**考查意图：**  
- 概率直觉是否准确
- 能否识别并避开反直觉陷阱
- 期望值的计算思维

### 4. 赌徒破产问题

**来源：** 阿里 / 字节（概率论基础）

**题目：**  
你和对手各有 50 元，每次抛公平硬币，正面你赢 1 元，反面你输 1 元，游戏直到一方破产为止。你最终获胜（对方先破产）的概率是多少？

**答案：** 50%

**解题思路：**

对于公平游戏（p = 1/2），赌徒破产定理给出：

$$P(\text{玩家获胜}) = \frac{\text{初始资金}}{\text{总资金}} = \frac{50}{50 + 50} = \frac{1}{2} = 50\%$$

若游戏不公平（p ≠ 1/2），公式变为：

$$P = \frac{1 - (q/p)^k}{1 - (q/p)^N}$$

其中 k 为初始资金，N 为总资金，p 为赢概率，q = 1-p。

**考查意图：**  
- 赌徒破产定理是否了解
- 能否快速判断公平游戏的对称性
- 数学建模能力

### 5. 毒药问题（二进制编码）

**来源：** 字节跳动 / 百度 / 谷歌（高频）

**题目：**  
1000 瓶水，其中 1 瓶有毒，喝了毒药 24 小时后死亡。只有 10 只小白鼠，如何在 24 小时内找出毒药？

**答案：** 用二进制编码，只需 1 轮测试（24 小时）。

**解题思路：**

10 只小白鼠可以表示 2^10 = 1024 种状态，足以覆盖 1000 瓶。

方法：
1. 将 1000 瓶水编号 1~1000，转为 10 位二进制（000000001 ~ 1111101000）
2. 第 i 只鼠喝掉所有**第 i 位为 1** 的瓶子的水
3. 24 小时后，死亡的鼠对应的二进制位组合，就是毒药的编号

**示例：**  
毒药是第 7 号（二进制 0000000111），则第 1、2、3 号鼠死亡，读出二进制 0000000111 = 7。

**考查意图：**  
- 二进制思维：用 n 位二进制表示 2^n 种状态
- 信息论：10 只鼠提供 10 bit 信息，恰好能区分 1000 个状态
- 将「生死结果」映射为「bit 信息」的创造性

## 第二类：逻辑推理

### 6. 三扇门（蒙提霍尔问题）

**来源：** 谷歌 / 微软 / 美团（反直觉概率经典）

**题目：**  
你面前有 3 扇门，一扇后面是新车，两扇后面是羊。你选了 1 号门，主持人知道哪扇是车，他打开了 3 号门（是羊）。此时你可以换到 2 号门，是否应该换？

**答案：** 应该换！换门获胜概率 = 2/3，不换 = 1/3。

**解题思路：**

| 初始选择 | 车的位置 | 不换结果 | 换门结果 |
|----------|----------|----------|----------|
| 门1 | 门1（车）| ✅ 赢 | ❌ 输 |
| 门1 | 门2（车）| ❌ 输 | ✅ 赢 |
| 门1 | 门3（车）| ❌ 输 | ✅ 赢 |

三种等概率情况，不换赢 1 次，换赢 2 次。

**直觉误区：** 「剩 2 扇门，各 50%」——这忽略了主持人的行为是**有信息量的**（他不会随机开门，他知道哪扇是车）。

**考查意图：**  
- 条件概率与贝叶斯思维
- 能否识别「信息不对称」改变了概率分布
- 是否敢于挑战直觉

### 7. 帽子颜色问题

**来源：** 谷歌 / 微软（博弈策略题）

**题目：**  
3 人站成一排（1 号在最后，能看到 2 人；2 号在中间，能看到 1 人；3 号在最前，什么都看不到）。每人随机戴红/蓝帽子，不能交流。三人同时猜自己帽子颜色，至少猜对 1 人即赢。如何制定策略使胜率达到 75%？

**答案：** 最优策略胜率 = 75%

**策略：**
- **1 号**（看到 2、3 号的帽子）：若两帽颜色**相同**，猜**相反色**；若两帽颜色**不同**，随便猜（或弃权）。
- **2 号**（看到 3 号的帽子）：策略类似，基于 3 号帽子颜色猜测。
- **3 号**（什么都看不到）：随便猜。

**分析：**  
共 8 种帽子组合，分析后可验证在 6/8 = 75% 的情况下至少有 1 人猜对。

**考查意图：**  
- 团队协作与策略制定
- 能否将「个体行为」转化为「系统最优」
- 博弈论基础

### 8. 12 个球称重问题

**来源：** 微软 / 谷歌 / 阿里（经典信息论题）

**题目：**  
12 个球，外观完全相同，其中 1 个重量异常（不知道是偏重还是偏轻）。用天平称 3 次，找出异常球并判断它偏轻还是偏重。

**答案：** 可以做到，采用三等分策略。

**解题思路（信息论角度）：**

每次天平称量有 3 种结果（左重/右重/平衡），3 次称量最多区分 3^3 = 27 种状态。  
12 个球 × 2（轻/重）= 24 种可能状态，恰好在 27 以内，理论可行。

**称法概要：**
1. **第 1 次**：4 vs 4（剩余 4 个放一边）
   - 平衡：异常球在剩余 4 个中
   - 不平衡：异常球在 8 个中（且知道哪边偏重）
2. **第 2 次**：根据第 1 次结果，进一步三等分排查
3. **第 3 次**：最终定位并判断轻重

（完整方案较复杂，面试中给出思路框架即可）

**考查意图：**  
- 信息论：每次称量的信息量（三进制）
- 系统性地排除可能性，而非随机试探
- 面试中展示思维过程比直接给答案更重要

### 9. 蓝眼睛问题（公共知识）

**来源：** 谷歌 / 字节（逻辑归纳经典）

**题目：**  
岛上有 100 人，所有人都有蓝眼睛，但没有镜子，也不能互相告知眼睛颜色，每人只知道别人的眼睛颜色。每天有船来，有蓝眼睛的人「如果知道自己有蓝眼睛」就必须当天离岛。一天外人来宣布：「这里至少有 1 人是蓝眼睛。」（这是公共知识）问：第几天有人离岛？

**答案：** 第 100 天，100 人同时离岛。

**解题思路（数学归纳法）：**

- **n = 1**：1 个蓝眼睛人，听到公告，知道是自己，第 1 天离岛。
- **n = 2**：每人看到对方 1 个蓝眼睛。第 1 天无人离岛（如果对方是唯一那个，对方应该走了）→ 第 2 天意识到对方没走，说明自己也是蓝眼睛 → 第 2 天两人同时离岛。
- **n = k**：第 k 天 k 人同时离岛（归纳成立）。
- **n = 100**：第 100 天 100 人同时离岛。

**关键：** 外人的公告创造了「公共知识」——不只是每个人知道，而是「每个人知道每个人知道……」这一无限层嵌套。

**考查意图：**  
- 数学归纳法的运用
- 「公共知识」与「私有知识」的区别
- 深层逻辑推理能力

### 10. 开关与灯问题

**来源：** 微软 / 百度（利用隐藏维度）

**题目：**  
房间外有 3 个开关，分别控制房间里的 3 盏灯，门关着看不到灯。只能进房间一次，如何判断哪个开关控制哪盏灯？

**答案：** 利用「热量」这个隐藏维度。

**操作步骤：**
1. 将开关 1 打开，等待 10 分钟，然后关掉
2. 将开关 2 打开，保持开着
3. 进入房间

**判断：**
- 亮灯 → 开关 2
- 灭灯但灯泡发热 → 开关 1
- 灭灯且灯泡不热（凉的）→ 开关 3

**考查意图：**  
- 打破思维定势：不局限于「亮/灭」二元状态，引入时间维度（热量）
- 创造性利用题目隐含的物理属性
- 第一性原理思维

## 第三类：数学思维

### 11. 水壶量水

**来源：** 谷歌 / 微软 / LeetCode 经典题

**题目：**  
只有 3 升和 5 升两个水壶，水源无限。如何量出恰好 4 升水？

**答案：** 按以下步骤操作：

| 步骤 | 操作 | 3升壶 | 5升壶 |
|------|------|-------|-------|
| 初始 | — | 0 | 0 |
| 1 | 5升壶装满 | 0 | 5 |
| 2 | 5升壶倒入3升壶 | 3 | 2 |
| 3 | 倒掉3升壶 | 0 | 2 |
| 4 | 5升壶(2升)倒入3升壶 | 2 | 0 |
| 5 | 5升壶装满 | 2 | 5 |
| 6 | 5升壶倒入3升壶(倒1升) | 3 | **4** ✅ |

**算法视角：** 这是一个 BFS 最短路径问题，状态为 (a, b) 表示两壶的水量，枚举所有合法转移直到出现目标状态。

**考查意图：**  
- BFS / 状态机思维
- 将实际问题抽象为图搜索
- 这道题在 LeetCode 有对应题目（LeetCode 365）

### 12. 数字黑洞 6174（卡普雷卡尔常数）

**来源：** 阿里 / 谷歌（数学好奇心考察）

**题目：**  
任取一个 4 位数（各位数字不全相同），将 4 个数字排成最大数和最小数，相减，将结果重复此操作，最终一定会收敛到 6174。为什么？

**示例：**  
9753 → 9753 - 3579 = 6174  
1000 → 1000 - 0001 = 0999 → 9990 - 0999 = 8991 → 9981 - 1899 = 8082 → 8820 - 0288 = 8532 → 8532 - 2358 = 6174

**解析：**  
6174 = 7641 - 1467，而 7641 - 1467 = 6174（自身是不动点）。  
通过穷举所有 10×10×10×10 = 10000 种 4 位数的操作路径，可以验证所有路径最终都指向 6174（约 7 步内必达）。  
严格数学证明较复杂，面试中能描述现象并尝试找规律即可。

**考查意图：**  
- 数学好奇心与探究欲
- 不要求证明，观察面试者能否快速探索、验证猜想
- 是否能从「不动点」角度切入分析

### 13. 无限旅馆（希尔伯特悖论）

**来源：** 谷歌 / 微软（无穷概念考察）

**题目：**  
一家旅馆有无限个房间（编号 1, 2, 3, …），且已经住满了无限位客人。  
- 又来了 1 位新客人，能否安置？  
- 又来了无限位新客人，能否安置？

**答案：**

**情况 1（+1 人）：**  
让每位客人从 n 号房移到 n+1 号房，1 号房就空出来了。✅

**情况 2（+∞ 人）：**  
让每位原客人从 n 号房移到 **2n** 号房（所有偶数房间），奇数号房间（1, 3, 5, …）全部空出，无限位新客人住进奇数房。✅

**深入思考：**  
甚至可以安置「无限多个无限团队」（用对角线法，康托尔的方法）。

**考查意图：**  
- 对「无穷大」概念的理解（∞ + 1 = ∞，∞ × 2 = ∞）
- 集合论基础，可数无穷的直觉
- 发散思维与抽象思维

### 14. 海盗分金（逆向博弈）

**来源：** 谷歌 / 微软 / 阿里（博弈论经典）

**题目：**  
5 个理性海盗（编号 1~5，1 号最强势），共有 100 枚金币。按规则：1 号先提分配方案，所有人投票，超过一半同意则执行，否则 1 号被扔海里，由 2 号提方案，以此类推。每个海盗优先保命，其次多拿金币，其次希望其他人死（同等条件下）。1 号应如何提方案？

**答案：** 1 号提案：自己拿 96 枚，3 号拿 1 枚，5 号拿 3 枚（或 1 号 96，3 号 2，5 号 2，具体版本略有差异）。

**逆向推导过程：**

- **只剩 4+5 号**：4 号提方案，只需自己同意（1/2 超半数不够，需要 >50% 即 2 票），4 号必须拉到 5 号，给 5 号 1 枚。4 号得 99，5 号得 1。
- **剩 3+4+5 号**：3 号需要 2 票。4 号知道 3 号死了自己能得 99，不容易收买。5 号在上种情况只能得 1，3 号给 5 号 2 枚就能买到票。3 号：98，5 号：2，4 号：0。
- **剩 2+3+4+5 号**：2 号需要 3 票（含自己）。4 号上种情况得 0，给 1 枚就能收买。3 号得 98，收买不了。5 号得 2，给 3 枚可以收买。2 号：97，4 号：1，5 号：3。（注：也可收买 3 和 4）
- **1+2+3+4+5（最终）**：1 号需要 3 票（含自己）。2 号必然反对（1 号死了他能得 97）。3 号上种情况得 0，给 1 枚收买。5 号上种情况得 3，给 4 枚收买或另选…

> **最终答案（一个常见版本）：** 1 号：96，2 号：0，3 号：1，4 号：0，5 号：3

**考查意图：**  
- 逆向归纳（Backward Induction）
- 博弈论与理性人假设
- 系统性分析复杂决策树的能力

### 15. 时钟夹角

**来源：** 美团 / 百度 / 阿里（细心程度考察）

**题目：**  
3 点 15 分时，时针和分针的夹角是多少度？

**答案：** 7.5 度（不是 90 度！）

**解题过程：**

- **分针位置**：15 分 = 360° × (15/60) = **90°**（指向 3 的方向）
- **时针位置**：
  - 时针每小时走 30°（360°/12）
  - 3 小时 + 15 分钟 = 3.25 小时
  - 时针位置 = 3.25 × 30° = **97.5°**
- **夹角** = 97.5° - 90° = **7.5°**

**误区：** 很多人看到「3:15」就说「分针指 3，时针指 3，夹角 0 度」——忽略了时针在每分钟也在走。

**考查意图：**  
- 细心与严谨性
- 能否注意到「时针不是整点停在数字上」这个细节
- 快速计算能力

## 第四类：前端场景智力题

### 16. 请求优先级调度

**来源：** 字节跳动 / 美团前端（工程思维）

**题目：**  
页面发了 100 个请求，浏览器同域名最多并发 6 个（HTTP/1.1 限制）。如果你是调度器，怎么设计保证关键资源（CSS/JS/首屏图片）最先加载？

**答案：**

**方案一：优先队列调度**
```javascript
class RequestScheduler {
  constructor(maxConcurrent = 6) {
    this.maxConcurrent = maxConcurrent;
    this.queue = []; // 按优先级排序的等待队列
    this.running = 0;
  }

  add(request, priority = 0) {
    this.queue.push({ request, priority });
    this.queue.sort((a, b) => b.priority - a.priority); // 高优先级排前面
    this.flush();
  }

  flush() {
    while (this.running < this.maxConcurrent && this.queue.length > 0) {
      const { request } = this.queue.shift();
      this.running++;
      request().finally(() => {
        this.running--;
        this.flush();
      });
    }
  }
}
```

**优先级定义：**
- P0（最高）：关键 CSS（阻塞渲染）
- P1：关键 JS（defer/async 中的核心逻辑）
- P2：首屏图片（LCP 相关）
- P3：懒加载图片、非关键资源

**其他优化手段：**
- 已在途的请求不取消（取消再重发更耗时）
- 相同接口的并发请求合并（singleflight）
- 用 `<link rel="preload">` 提前声明关键资源
- Service Worker 缓存 + 离线策略

- 优先队列的实际工程应用
- 对浏览器并发限制的理解
- 性能优化的系统性思维
