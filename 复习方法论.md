# 前端知识复习方法论

> 整理时间：2026-03-31  
> 适用场景：面试备战 / 知识巩固 / 跳槽准备

---

## 一、核心原则：输出驱动复习

> **「能讲出来」比「看懂了」重要 10 倍**

读完一个知识点后，立刻合上笔记，用自己的话说一遍。能说清楚 = 真的懂了。不能说清楚 = 还需要再看。

---

## 二、三轮复习法

### 第一轮：扫盲（1-2 天）

**目标**：建立全局地图，找出不确定的点

- 每个模块快速过一遍
- 不懂的地方打 `❓`，暂时跳过
- 不要卡在某个知识点超过 5 分钟

**适合材料**：`复习清单.md` → 按模块速读

---

### 第二轮：深挖（2-3 天）

**目标**：把 ❓ 标记的知识点搞清楚，形成口头表达能力

- 专注在第一轮打了 ❓ 的内容
- 每个知识点用「费曼技巧」（见下方）过一遍
- 重点背记：EventLoop、原型链、Promise、Vue 响应式、HTTP缓存 等必考点

**方法**：「费曼技巧」——假装在给不懂的人解释这个概念

---

### 第三轮：手写+模拟（1-2 天）

**目标**：验证掌握程度，发现最后的盲点

- 把 `13-面试手写/` 里的题目一道道手写（不看答案）
- 找朋友或贾维斯做 1-2 次模拟面试
- 自己录音说一遍某个复杂概念（听自己说话能发现逻辑漏洞）

---

## 三、具体技巧

### 🧠 费曼技巧（深度理解必备）

1. 选一个概念（如「闭包」）
2. 写下来 / 说出来，假装在教一个不懂的人
3. 遇到说不清楚的地方 → 回去看资料补充
4. 用最简单的语言重新解释

> 例：闭包 = 函数记住了它被创建时的作用域，即使这个作用域已经消失了。

---

### 📦 主动回忆（Active Recall）

❌ 错误做法：反复读笔记  
✅ 正确做法：合上笔记，回忆刚才读的内容，写/说出来

研究表明主动回忆的记忆效率是被动阅读的 **3-4 倍**。

**落地方式**：
- 读完一个知识点 → 合上 → 口头复述
- 第二天早上 → 回忆昨天学了什么
- 用 `spaced-review` 闪卡系统（间隔复习）

---

### ⏱ 番茄工作法（保持专注）

- 25 分钟专注 + 5 分钟休息 = 1 个番茄
- 连续 4 个番茄后，休息 20-30 分钟
- 复习任务按番茄数估时（如：「TypeScript 高级特性」= 2 个番茄）

---

### 🗂 康奈尔笔记法（记笔记时用）

```
┌─────────────────┬───────────────────────────────────┐
│   关键词/问题   │           详细内容笔记             │
│                 │                                   │
│  EventLoop      │  宏任务、微任务、调用栈 ...       │
│  是什么？       │                                   │
│  为什么重要？   │                                   │
│                 │                                   │
├─────────────────┴───────────────────────────────────┤
│  总结：用 1-2 句话概括这一页的核心                  │
└──────────────────────────────────────────────────────┘
```

---

### 🔁 间隔复习（Spaced Repetition）

遗忘曲线：学完后 1 天、3 天、7 天、14 天各复习一次，记忆效果最好。

| 复习时间 | 记忆保留率 |
|---------|-----------|
| 不复习 | 1 天后 ~40% |
| 复习 1 次 | ~60% |
| 按间隔复习 4 次 | ~90%+ |

**工具**：项目中的 `spaced-review` skill（基于 SM-2 算法，推荐面试结束后启用）

## 四、前端面试专项策略

### 答题结构（STAR/总分总）

**推荐格式**：
1. **是什么**（定义/概念）
2. **为什么**（原理/机制）
3. **怎么用**（场景/代码示例）
4. **注意点**（坑/边界/优化）

> 例如被问「闭包是什么」：
> - 是什么：函数能访问其词法作用域中的变量，即使函数在其他地方被调用
> - 为什么：JS 使用词法作用域，函数创建时就确定了作用域链
> - 怎么用：防抖节流、模块化封装私有变量、React hooks 实现
> - 注意：内存泄漏风险，循环中闭包的经典陷阱

---

### 高频考点优先级

```
🔴 P0（必须背熟）：
  - EventLoop / 宏任务微任务
  - 原型链 / 继承
  - 闭包 / 作用域
  - Promise / async-await 原理
  - Vue 响应式原理 / diff 算法
  - HTTP 缓存 / 状态码 / HTTPS
  - 浏览器渲染流程 / 重排重绘

🟡 P1（理解+能说）：
  - TypeScript 泛型 / 类型体操
  - Webpack 构建流程
  - React Fiber / Hooks 原理
  - 跨域方案 / 安全（XSS/CSRF）
  - 性能优化指标与方案

🟢 P2（了解即可）：
  - Node.js 事件循环（与浏览器区别）
  - 微前端架构
  - WebSocket / WebRTC
```

---

### 手写题策略

1. **先说思路**，再写代码（让面试官知道你在想什么）
2. **说边界条件**（null 怎么处理、空数组怎么处理）
3. **不确定时**，主动说「我先实现核心逻辑，边界情况我们最后补」
4. **写完要测试**，用几个例子走一遍

> 手写题仓库：`12-面试手写/`

## 六、推荐复习顺序

```
Week 1：基础扫盲
  Day 1-2：HTML/CSS + JS 基础（作用域/闭包/原型链）
  Day 3-4：异步 Promise/EventLoop + ES6+
  Day 5-6：Vue/React 核心原理
  Day 7：复习 + 查漏补缺

Week 2：深挖 + 手写
  Day 8-9：网络/浏览器原理（HTTP/渲染流程）
  Day 10-11：工程化（Webpack/Vite）+ TS
  Day 12-13：手写题集中练习
  Day 14：模拟面试 × 1

Week 3：冲刺
  Day 15-17：高频面试题口头背诵
  Day 18-19：项目亮点整理 + 自我介绍打磨
  Day 20-21：模拟面试 × 2-3
```
