## 一、布局类

### 1. Anchor Positioning（锚点定位）⭐⭐⭐

**背景**：Popper.js 是前端最流行的浮动层定位库，过去 8 年靠 JS 计算位置、碰撞检测、实现复杂。Anchor Positioning 让这一切原生化。

**解决什么问题**：Tooltip、下拉菜单、浮动卡片等"相对于某个元素定位"的场景，不再需要 JS 库，浏览器原生支持空间判断和自动翻转。

```css
/* 定义锚点 */
.trigger-btn {
  anchor-name: --my-anchor;
}

/* 浮动层定位 */
.tooltip {
  position: absolute;
  position-anchor: --my-anchor;
  inset-area: top center; /* 在锚点上方居中 */
  position-try-fallbacks: flip-block; /* 空间不足时自动翻转 */
}
```

**自动翻转示例**：

```css
.dropdown {
  position: absolute;
  position-anchor: --dropdown-trigger;
  inset-area: bottom span-right; /* 锚点下方，向右展开 */
  position-try-fallbacks: flip-block, flip-inline; /* 优先向下，不够则翻上 */
}
```

**追问**：
- `anchor()` 函数和 `inset-area` 有什么区别？→ `anchor()` 精确指定锚点的边；`inset-area` 声明"方位区域"，更简洁
- `position-try-fallbacks` 支持哪些策略？→ `flip-block`、`flip-inline`、`same-block`、`same-inline`，以及自定义 `@position-try` 块

### 2. CSS Masonry（瀑布流布局）

**背景**：瀑布流（Pinterest 风格）是 2010 年代初兴起的布局模式，过去只能用 JS（jQuery Masonry / CSS columns hack）实现。2026 年 CSS 终于原生支持。

**解决什么问题**：高度不一的元素自动填充到最短列，无需 JS 计算。

**浏览器状态（2026.04）**：Safari 18 正式支持（`display: grid-lanes`），Chrome 和 Firefox 仍是实验性。

```css
/* Safari 18+ 语法 */
.masonry-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: masonry;
  gap: 16px;
}

/* Chrome flag 语法（可能变化）*/
@supports (grid-template-rows: masonry) {
  .masonry-grid {
    grid-template-rows: masonry;
  }
}
```

**降级方案**：未支持时退回普通 grid + JS Masonry 库。

---

### 3. CSS Subgrid（子网格）⭐⭐⭐

**背景**：CSS Grid 解决了页面级布局，但"卡片列表中每个卡片内部的对齐"始终是难题。子元素无法参与父网格的轨道，所以每个卡片内容只能各自为政。

**解决什么问题**：子网格让网格项继承父网格的行/列轨道，实现多行卡片内容的完美对齐。

```css
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: auto auto auto; /* 3行：标题、内容、底部 */
  gap: 24px;
}

.card {
  grid-row: span 3;
  display: grid;
  grid-template-rows: subgrid; /* 继承父网格的3行轨道 */
}

.card__header { /* 自动对齐到父网格第1行 */ }
.card__content { /* 自动对齐到父网格第2行 */ }
.card__footer { /* 自动对齐到父网格第3行，即使内容高度不同 */ }
```

**适用场景**：商品卡片列表、仪表盘面板、表格行分组。

## 二、交互与弹层

### 4. interestfor + Popover API（纯 CSS hover 弹层）⭐⭐⭐

**背景**：过去 tooltip 需要 JS 监听 `mouseenter/mouseleave`，或者依赖第三方库。Popover API（Chrome 117）带来了声明式弹层，`interestfor` 补充了 hover 触发能力。

**解决什么问题**：hover 交互完全无需 JavaScript，声明式实现。

```html
<button interestfor="my-tooltip">悬停我</button>
<div id="my-tooltip" popover>提示内容</div>
```

**Popover 完整体系**：

| 属性/方法 | 触发方式 | 用途 |
|-----------|---------|------|
| `popover` | 声明式 | 标记为弹层元素 |
| `popovertarget` | 点击触发 | 按钮的 `popovertarget="my-popover"` |
| `interestfor` | hover 触发 | 声明对哪个 popover 感兴趣 |
| `showPopover()` | JS 手动 | 手动显示 |
| `hidePopover()` | JS 手动 | 手动隐藏 |

**追问**：`popover` 属性的值有哪些？→ `auto`（轻量，外部点击关闭）和 `hint`（不关闭已有 `auto` popover，Safari Firefox 支持中）。

---

### 5. dialog.closedBy（原生弹窗关闭策略）⭐⭐⭐

**背景**：`<dialog>` 的 `showModal()` 显示了 backdrop 蒙层，但点击蒙层不会关闭——这是十年的交互痛点。必须写 JS hack 判断点击位置。

**解决什么问题**：一个 HTML 属性，`closedby="any"` 直接让点击蒙层生效，零 JS。

```html
<!-- 点击蒙层或按 Esc 都关闭 -->
<dialog closedby="any">
  <h2>标题</h2>
  <p>内容</p>
</dialog>
<button onclick="document.querySelector('dialog').showModal()">打开</button>
```

**三个值**：

| 值 | 蒙层点击 | Esc | JS .close() |
|---|---|---|---|
| `any` | ✅ 关闭 | ✅ 关闭 | ✅ 关闭 |
| `closerequest` | ❌ | ✅ | ✅ |
| `none` | ❌ | ❌ | ✅ |

**追问**：
- `show()` 和 `showModal()` 的区别？→ `showModal()` 有蒙层、阻断背后交互、支持 Esc 关闭
- `::backdrop` 怎么用？→ `dialog::backdrop { background: rgba(0,0,0,0.5); }`

### 6. Custom Highlight API（编程式文本高亮）

**背景**：过去给搜索结果高亮需要操作 DOM（插入 `<mark>` 标签），改变现有结构，或者依赖 Selection API 做视觉覆盖但不可交互。

**解决什么问题**：直接用 CSS 高亮任意文本范围，不修改 DOM，不干扰无障碍。

```javascript
CSS.highlights.set('search', new Highlight(range1, range2));
```

```css
::highlight(search) {
  background-color: #ffeb3b;
  color: #000;
}
```

**典型场景**：搜索结果高亮、段落内语法高亮、实时字幕同步。

## 三、动画

### 7. Scroll-Driven Animations（滚动驱动动画）⭐⭐⭐

**背景**：实现滚动触发动画（如阅读进度条、视差效果）过去需要 `IntersectionObserver` + `window.scroll` 监听，性能差且代码复杂。CSS Scroll-Driven Animations 用纯 CSS 解决。

**解决什么问题**：绑定动画进度到滚动位置，合成器线程执行，主线程繁忙也不掉帧。

```css
/* 阅读进度条（基于 viewport 滚动进度）*/
.progress-bar {
  animation: grow linear forwards;
  animation-timeline: scroll(root);
}

@keyframes grow {
  from { transform: scaleX(0); }
  to { transform: scaleX(1); }
}
```

```css
/* 视差效果（基于元素在视口中的可见度）*/
.hero {
  animation: parallax linear forwards;
  animation-timeline: view();
}
```

**浏览器支持**：Chrome 115+、Firefox 132+、Safari 18+ — 2026 年已全浏览器可用。

---

### 8. @starting-style（进入动画起点）

**背景**：CSS 动画只能定义"从哪开始"，但元素进入页面的"起始状态"无法声明。`@starting-style` 解决了这个问题，让进入动画有明确起点。

**解决什么问题**：元素从 `display: none` 或 `visibility: hidden` 切换时，动画没有清晰的 from 状态，导致跳帧。

```css
@starting-style {
  .card {
    opacity: 0;
    transform: translateY(20px);
  }
}

.card {
  animation: slide-in 0.3s ease-out forwards;
}

@keyframes slide-in {
  to { opacity: 1; transform: translateY(0); }
}
```

**典型场景**：模态框打开动画、Toast 通知进入、列表项入场动画。

## 四、颜色

### 9. LCH Colors（感知均匀颜色）

**背景**：HSL 是设计师友好的，但它的感知均匀性差——HSL 里的"亮度 L=50%"在不同色相下实际明暗差异很大。LCH 更符合人眼感知。

**解决什么问题**：程序化生成配色时，LCH 可以安全地调整亮度而不意外影响饱和度。

```css
/* LCH: Lightness / Chroma / Hue */
--primary: lch(60% 80 270); /* 亮紫色 */

--primary-lighter: lch(from var(--primary) calc(l + 15%) c h); /* 亮度+15%，色相色度不变 */
--primary-more-saturated: lch(from var(--primary) l calc(c * 1.2) h);
```

**追问**：`from var(--primary)` 是什么意思？→ Relative Color Syntax（CSS Color Level 5），从已有颜色派生，继承色相/色度/亮度进行运算。

## 五、排版与单位

### 10. text-wrap: balance / pretty（智能换行）

**背景**：英文文本默认按单词边界换行，可能导致一行最后只剩一个单词（孤儿行）或一行开头只剩一个单词（寡妇产），排版不美观。

**解决什么问题**：自动优化断行位置，减少孤儿寡妇行。

```css
h1 { text-wrap: balance; }  /* 视觉平衡，限制换行次数 */
p  { text-wrap: pretty; }   /* 减少孤儿寡妇行，优化阅读 */
```

---

### 11. lh / rlh 单位（基于行高的单位）

**背景**：过去 CSS 没有基于行高的相对单位。如果想让一个元素的 `padding-top` 等于当前行高，需要 `line-height * n` 的 JS 计算。

**解决什么问题**：直接用 `lh` 单位引用当前 `line-height` 值。

```css
.box {
  padding-top: 1lh; /* 上内边距 = 一行文字的高度 */
  line-height: 1.5;
}
```

## 六、作用域

### 12. @scope（作用域样式）

**背景**：CSS 的全局性是一把双刃剑。`@layer` 解决了层叠优先级问题，但没有解决"限定选择器作用范围"的问题。

**解决什么问题**：把选择器的作用域限定在特定 DOM 子树，不需要 BEM 那种长命名。

```css
/* h2 只在 .card 内生效 */
@scope (.card) {
  h2 { font-size: 1.25rem; }
}

/* 排除某些范围 */
@scope (.card) to (.actions) {
  /* h2 在 .card 内生效，但在 .actions 内不生效 */
  h2 { color: #333; }
}
```

**与 @layer 的区别**：
- `@layer` 控制优先级/层叠顺序
- `@scope` 控制选择器的作用范围

## 七、面试高频问题汇总

### CSS 2026 面试三问

**Q1：Anchor Positioning 和 Floating UI / Popper.js 的区别？**
- 库提供：完整的功能（定位+动画+无障碍），成熟稳定，跨浏览器
- 原生提供：无依赖，浏览器原生优化，但功能集较小
- **结论**：简单场景用原生，复杂交互（智能定位、虚拟元素、超出边界检测）仍需库

**Q2：dialog.closedBy 的三个值和最佳使用场景？**
- `any`：轻量通知弹窗（点击外部快速消除）
- `closerequest`：表单弹窗（防止误点丢失数据）
- `none`：确认操作弹窗（强制用户明确选择）

**Q3：Scroll-Driven Animations 和 IntersectionObserver 实现滚动动画的区别？**
- `IntersectionObserver`：JS API，检测元素可见性，动画由 JS 或 CSS 控制
- `Scroll-Driven Animations`：纯 CSS，动画进度直接绑定滚动进度，性能更好（合成器线程）
- **进阶追问**：为什么 Scroll-Driven Animations 性能更好？→ 动画完全在合成器线程执行，不触发主线程
