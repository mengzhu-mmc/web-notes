## 一、性能指标（Core Web Vitals）

Google 定义的核心性能指标：

| 指标 | 全称 | 含义 | 良好标准 |
| --- | --- | --- | --- |
| LCP | Largest Contentful Paint | 最大内容绘制（首屏主要内容加载完成） | < 2.5s |
| FID | First Input Delay | 首次输入延迟（用户首次交互到响应） | < 100ms |
| CLS | Cumulative Layout Shift | 累积布局偏移（页面元素意外移动） | < 0.1 |
| FCP | First Contentful Paint | 首次内容绘制 | < 1.8s |
| TTFB | Time to First Byte | 首字节时间（服务器响应速度） | < 800ms |

```js
// 使用 Performance API 采集指标
// FCP
new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (entry.name === 'first-contentful-paint') {
      console.log('FCP:', entry.startTime);
    }
  }
}).observe({ type: 'paint', buffered: true });

// LCP
new PerformanceObserver((list) => {
  const entries = list.getEntries();
  const last = entries[entries.length - 1];
  console.log('LCP:', last.startTime);
}).observe({ type: 'largest-contentful-paint', buffered: true });
```

## 二、网络层优化

### 减少请求数量

```html
<!-- 1. 合并 CSS/JS 文件（Webpack 打包） -->
<!-- 2. CSS Sprites（雪碧图）合并小图标 -->
<!-- 3. 内联小资源（base64 图片、内联 CSS） -->
<!-- 4. 使用字体图标替代图片图标 -->
```

### 减少请求体积

```
1. 开启 Gzip/Brotli 压缩（服务端配置）
   - Gzip：压缩率约 70%
   - Brotli：比 Gzip 再压缩 20-26%

2. 图片压缩与格式选择
   - WebP：比 JPEG 小 25-35%，比 PNG 小 26%
   - AVIF：比 WebP 再小 20%（兼容性较差）
   - SVG：矢量图，适合图标
   - 使用 <picture> 标签做格式降级

3. 代码压缩（Webpack TerserPlugin）
   - 删除注释、空白
   - 变量名混淆
   - Tree Shaking（删除未使用代码）
```

### 缓存策略

```
强缓存（不发请求）：
  Cache-Control: max-age=31536000  // 1年
  适用：带 hash 的静态资源（main.abc123.js）

协商缓存（发请求，可能返回 304）：
  ETag / Last-Modified
  适用：HTML 文件（不能强缓存，否则更新不及时）

Service Worker 缓存：
  可以完全控制缓存策略，支持离线访问
```

### CDN

```
1. 将静态资源部署到 CDN
2. CDN 节点分布全球，用户访问最近的节点
3. CDN 域名与主域名不同，避免携带 Cookie
4. 开启 HTTP/2（多路复用，减少连接数）
```

## 三、渲染层优化

### 重排（Reflow）和重绘（Repaint）

```
重排（回流）：元素的几何属性变化，需要重新计算布局
  触发：width/height/margin/padding/position/display 等

重绘：元素外观变化，不影响布局
  触发：color/background/visibility/box-shadow 等

重排一定触发重绘，重绘不一定触发重排
重排的代价 >> 重绘的代价
```

**避免重排的技巧：**

```js
// ❌ 频繁读写 DOM，触发多次重排
for (let i = 0; i < 100; i++) {
  element.style.left = element.offsetLeft + 1 + 'px'; // 读写交替
}

// ✅ 批量读，批量写
const left = element.offsetLeft; // 一次读
for (let i = 0; i < 100; i++) {
  element.style.left = left + i + 'px'; // 批量写
}

// ✅ 使用 DocumentFragment 批量操作 DOM
const fragment = document.createDocumentFragment();
for (let i = 0; i < 100; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  fragment.appendChild(li);
}
list.appendChild(fragment); // 只触发一次重排

// ✅ 使用 CSS transform 代替 top/left（不触发重排）
// ❌ element.style.left = '100px';
// ✅ element.style.transform = 'translateX(100px)';

// ✅ 使用 will-change 提示浏览器创建独立图层
.animated {
  will-change: transform;
}
```

### 图片懒加载

```html
<!-- 原生懒加载（现代浏览器支持） -->
<img src="image.jpg" loading="lazy" alt="..." />

<!-- IntersectionObserver 实现 -->
<img data-src="image.jpg" class="lazy" alt="..." />
```

```js
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      observer.unobserve(img); // 加载后停止观察
    }
  });
}, {
  rootMargin: '200px' // 提前 200px 开始加载
});

document.querySelectorAll('img.lazy').forEach(img => observer.observe(img));
```

## 四、JavaScript 优化

### 长任务优化

```js
// 问题：长任务阻塞主线程，导致页面卡顿
function heavyTask() {
  // 处理 10000 条数据，耗时 500ms
  for (let i = 0; i < 10000; i++) {
    processItem(data[i]);
  }
}

// 方案一：分批处理（时间切片）
function processInChunks(data, chunkSize = 100) {
  let index = 0;
  function processChunk() {
    const end = Math.min(index + chunkSize, data.length);
    for (; index < end; index++) {
      processItem(data[index]);
    }
    if (index < data.length) {
      // 让出主线程，下一帧继续
      requestAnimationFrame(processChunk);
      // 或者：setTimeout(processChunk, 0);
    }
  }
  processChunk();
}

// 方案二：Web Worker（将计算移到后台线程）
const worker = new Worker('worker.js');
worker.postMessage({ data: largeData });
worker.onmessage = (e) => {
  console.log('计算结果：', e.data);
};
```

### 防抖节流（见 防抖与节流.md）

### 避免内存泄漏

```js
// 1. 及时清除定时器
const timer = setInterval(fn, 1000);
// 组件卸载时
clearInterval(timer);

// 2. 及时移除事件监听
const handler = () => {};
element.addEventListener('click', handler);
// 组件卸载时
element.removeEventListener('click', handler);

// 3. 避免意外的全局变量
function fn() {
  // ❌ 没有 var/let/const，变成全局变量
  leak = 'this is a leak';
}

// 4. 使用 WeakMap/WeakSet 存储 DOM 引用
const cache = new WeakMap(); // DOM 节点被移除后，缓存自动清理
```

## 六、首页白屏优化专题

白屏的核心原因：大量 JS 需要下载/解析/执行；纯 CSR 首屏需等 JS 执行完才有内容；关键资源加载慢。

### 6大优化类别

**1. 资源加载优化（最常见最有效）**
- 压缩资源：JS/CSS/HTML 压缩、图片 WebP/AVIF
- 路由懒加载、组件懒加载、图片懒加载
- CDN 加速静态资源
- 减少请求数：雪碧图、内联小资源（base64）

**2. 构建与打包优化**
- Code Splitting：按路由/组件拆包
- Tree Shaking：剔除未使用代码
- 替换大库（Moment → Day.js）、按需引入
- Vite 预构建、DLL Plugin 缓存依赖

**3. 渲染层面优化（解决白屏最直接）**
- **SSR（服务端渲染）**：首屏直接返回带内容 HTML，彻底消除白屏
- **SSG（静态站点生成）**：构建时生成 HTML，首屏秒出
- **骨架屏 Skeleton**：快速占位，降低用户感知白屏时间
- `content-visibility: auto`：跳过屏外内容渲染
  ```css
  .list-item {
    content-visibility: auto;
    contain-intrinsic-size: 1px 300px; /* 预估高度，防布局抖动 */
  }
  ```

**4. 网络与缓存优化**
- HTTP 强缓存 + 协商缓存（静态资源 max-age + hash，HTML no-cache）
- Service Worker 离线缓存
- DNS 预解析：`<link rel="dns-prefetch">`
- 关键资源预加载：`<link rel="preload">`

**5. 运行时与执行优化（JS 阻塞）**
- `defer`/`async` 异步加载 JS，不阻塞 HTML 解析
- 减少长任务 Long Task，使用 Web Worker
- 避免渲染阻塞：减少同步 DOM 操作

**6. 首屏指标优化（可量化）**
- LCP（最大内容绘制）：优先加载首屏图片/内容 < 2.5s
- FCP（首次内容绘制）：尽快让页面出现任何内容 < 1.8s
- TTI（可交互时间）：尽快让页面能点能用

### 双维度优化思路

**静态资源**：更小（压缩/WebP/剔除冗余）、更少（合并/Tree Shaking）、更快（CDN/缓存）、更稳（哈希 + 强缓存）

**网络请求**：少发（合并/本地缓存）、早发（DNS预解析/预连接/preload）、快回（CDN/HTTP2/gzip）、并行（HTTP2多路复用）

## 七、性能优化检查清单

```
网络层：
□ 开启 Gzip/Brotli 压缩
□ 静态资源上 CDN
□ 合理设置缓存策略（HTML 协商缓存，JS/CSS 强缓存+hash）
□ 使用 HTTP/2
□ 图片使用 WebP 格式
□ 关键资源 preload，非关键资源 prefetch

渲染层：
□ 避免强制同步布局（读写分离）
□ 使用 transform/opacity 做动画（不触发重排）
□ 图片懒加载
□ 虚拟列表（大数据量）

JavaScript：
□ 代码分割 + 懒加载
□ Tree Shaking
□ 防抖/节流
□ Web Worker 处理密集计算
□ 避免内存泄漏

React 专项：
□ React.memo 避免不必要渲染
□ useMemo/useCallback 缓存
□ 状态下移（State Colocation）
□ 合理使用 key
```
