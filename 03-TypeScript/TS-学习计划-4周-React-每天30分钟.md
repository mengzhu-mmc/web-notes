# TypeScript 学习计划（React 前端｜TS 2/10｜每天 30 分钟｜每周 3 小时）

> 目标：**4 周内把 TS 用进真实 React 页面开发**，能写、能看懂报错、能逐步减少 `any`。
>
> 节奏：每天 30 分钟，周 6 天（建议周日休息/复盘）。
>
> 原则：只学在 React 项目里马上用得上的 20%，**不学类型体操**。

---

## 总览

- **第 1 周：React 写得下去**（props / state / event 三件套）
- **第 2 周：表单与接口响应**（最容易脏的地方先管住）
- **第 3 周：泛型入门**（只学最实用的）
- **第 4 周：工程习惯**（让 TS 变成长期收益）

每周都有“产出物”，建议直接做一个小 PR 或至少本地落地到一个真实组件。

---

## 第 1 周：React 写得下去（props / state / event）

**本周目标**：把一个现有组件迁成 TSX，并且做到 **0 个 `any`**（实在绕不过先用 `unknown` 顶住）。

### 每天 30 分钟安排（周 6 天）
- **D1**：基础类型 + 数组/对象 + 函数参数/返回值标注
- **D2**：`type` vs `interface`（规则：对象形状用 `interface`，联合/工具用 `type`）
- **D3**：Props 类型：必填/可选（`?:`）/ 默认值
- **D4**：事件类型：`onChange`/`onClick`，理解 `e.currentTarget`
- **D5**：`useState` 推断：什么时候需要显式泛型
- **D6**：迁移 1 个真实组件到 TSX（优先从“表单/列表/弹窗”里选一个）

### 本周产出物
- [ ] `ComponentA.tsx`（或你项目中任意一个组件）完成迁移
- [ ] 迁移过程中遇到的 3 个典型报错 + 你最终怎么修复（记录在文件末尾即可）

---

## 第 2 周：表单与接口响应（边界层先稳住）

**本周目标**：让表单 state、接口返回从“到处乱飞”变成“有结构、有收敛”。

### 每天 30 分钟安排
- **D1**：联合类型 `A | B` + 空值处理策略（`undefined/null`）
- **D2**：类型收窄：`typeof` / `in`（能看懂报错并修）
- **D3**：表单 state 建模：字段 + 错误 + 提交状态
- **D4**：API Result 模板（判别联合）：`Success<T> | Fail`
- **D5**：写 2 个 type guard：比如 `isSuccess(res)`、`isFail(res)`
- **D6**：把你项目里 1 个真实接口调用改造为 Result 模板 + 收窄使用

### 本周产出物（建议直接落在 `types/` 或 `api/` 目录）
- [ ] `Result<T>` 类型定义
- [ ] `isSuccess()` type guard
- [ ] 1 个真实接口调用从“裸 data”改成“Result<T> + 收窄”

---

## 第 3 周：泛型入门（只学最实用的 20%）

**本周目标**：能写最常用的泛型工具函数，并在组件里真实使用。

### 每天 30 分钟安排
- **D1**：泛型函数：`function f<T>(x: T): T`
- **D2**：泛型约束：`K extends keyof T`
- **D3**：实现 `pick<T, K extends keyof T>()`
- **D4**：实现 `safeGet<T, K extends keyof T>()`
- **D5**：工具类型：`Pick/Omit/Partial`（理解用途 + 何时用）
- **D6**：把你项目里 1 个 util 重写成泛型版（并替换调用点）

### 本周产出物
- [ ] `utils/typed.ts`（至少 3 个泛型工具函数）
- [ ] 组件里真实调用（不要只写不落地）

---

## 第 4 周：工程习惯（让 TS 产生长期收益）

**本周目标**：形成“边写边收敛”的节奏：TS 更严格一点，但不折磨自己。

### 每天 30 分钟安排
- **D1**：`tsconfig` 最小认知：重点就一个 `strict`（先开，再逐步修）
- **D2**：`any` 清理策略：优先改“边界层”（API/表单/第三方库）
- **D3**：第三方库类型问题：写 adapter/封装，不在业务里到处 `as`
- **D4**：组件复用：通用 Props 类型（列表/表格/弹窗选一个）
- **D5**：整理“TS 自救清单”（你项目里最常见的 10 个问题以内）
- **D6**：做一次小型迁移：2-3 个组件 + 1 个接口层文件（一个小 PR 规模）

### 本周产出物
- [ ] `TS 自救清单`（10 条以内，放到项目 README 或本文件末尾）
- [ ] 一个小 PR / 一次可回滚的迁移提交（建议）

---

## 学习中最该避的坑（照做就能少走弯路）

1. **不学类型体操**（先把联合类型/收窄/泛型入门吃透）
2. **别用 `as` 解决一切**：优先改数据结构/写收窄函数
3. **遇到第三方类型报错**：先包一层 adapter，别把妥协扩散到业务代码
4. **`any` → `unknown` 是进步**：用 `unknown` 逼自己写收窄
